llama.cpp Release b8908 Patches Heap-Buffer-Overflow in Server

The llama.cpp project has issued a security fix for a heap-buffer-overflow vulnerability in its server component, tracked as CVE-2026-21869.

GitHub llama.cpp Releases 2026-04-23 23:18 UTC Key: github-llamacpp-releases::identity::b8908 Confidence: strong Mode: claude

Article body

Release b8908 of llama.cpp addresses a heap-buffer-overflow flaw in the server's `update_slots()` context-shift loop. The bug was triggered when a client submitted a negative value for the `n_discard` parameter in JSON, causing out-of-bounds memory access. The vulnerability carries a CVSS score of 8.8 and is classified as CWE-787 (out-of-bounds write). The fix clamps `n_discard` to a non-negative value at the JSON parse boundary, rejecting malformed input before it reaches the affected loop.

The release also normalizes `n_discard` handling so that zero-triggered auto-discard continues to work as expected. The patch notes reference the upstream advisory GHSA-8947-pfff-2f3c, and the commit was co-authored by Georgi Gerganov, the primary maintainer of the project.

Pre-built binaries for this release are available across a wide range of platforms: macOS (Apple Silicon, Intel), iOS, Linux (Ubuntu on multiple architectures including s390x, and with Vulkan/ROCm/OpenVINO/SYCL support), Android arm64, Windows (CPU, CUDA 12 and 13, Vulkan, SYCL, HIP), and openEuler on both x86 and aarch64 with ACL Graph acceleration variants.

Why this matters

  • A CVSS 8.8 score puts this vulnerability in the high-severity range, meaning it is exploitable over the network without special privileges and could lead to code execution or memory corruption.
  • The bug is in the server component, which is the entry point for remote clients sending inference requests, making any publicly accessible llama.cpp server a potential target.
  • The release ships a large matrix of platform-specific binaries, reflecting llama.cpp's broad deployment across CPUs, GPUs, and mobile/edge devices, all of which now receive the patch.
