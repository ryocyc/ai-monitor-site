Dify v1.13.3 Lands with Workflow, Streaming, and Knowledge Retrieval Fixes

Dify has shipped v1.13.3, a patch focused on stability and correctness across workflow execution, real-time streaming, and knowledge base retrieval.

GitHub Dify Releases 2026-04-22 12:43 UTC Key: github-dify-releases::identity::github-dify-releases-dify-release-feed-no Confidence: strong Mode: claude

Article body

The release introduces variable-reference support for model parameters in LLM, Question Classifier, and Variable Extractor nodes, allowing users to wire dynamic inputs more cleanly in workflow graphs. This comes by way of contributor @scdeng in pull request #33082.

On the bug-fix side, the team addressed StreamsBroadcastChannel replay and concurrency issues that were destabilising frontend/backend event delivery (#34030, #34061). The workflow editor also now prevents pasted nodes from carrying over Loop/Iteration metadata, and blocks HumanInput nodes from being pasted into invalid containers.

Knowledge retrieval received several targeted fixes: citation metadata is now preserved in web responses, crashes when dataset icon metadata is absent have been resolved, hit-count query filtering is corrected, and indexed document chunk previews are restored. An upgrade note alerts users to a pre-existing change in default Python and Node.js paths for Sandbox that may require a manual config update when coming from v1.13.2.

Why this matters

  • Variable-reference support in core workflow nodes gives non-technical users more flexible workflow design without hardcoding model parameters.
  • Streaming reliability fixes close a gap that could cause event delivery inconsistencies between frontend and backend during active runs.
  • The cluster of knowledge retrieval fixes — citation preservation, crash avoidance, and chunk preview restoration — directly improves the accuracy of answers generated from vectorised content.
