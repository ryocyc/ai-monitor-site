last30days-skill v3.1.0 Adds X API v2 OAuth2 Backend and Competitor Auto-Discovery

The last30days-skill intelligence-gathering tool ships version 3.1.0, consolidating five point releases and introducing an official X API v2 backend alongside a new competitor comparison flag.

Article body

The open-source last30days-skill tool, used for automated competitive and market intelligence tracking, has released version 3.1.0. The release consolidates changes from versions 3.0.10 through 3.0.14 and adds two notable new capabilities. A contributor under the handle @chaosreload contributed an `xurl` CLI as an alternative X (formerly Twitter) search backend using the official API v2 via OAuth2, replacing any prior unofficial scraping approach. Separately, maintainer @mvanhorn added a `--competitors` flag that enables automatic discovery and comparison fan-out across competitor entities, along with a new "vs mode" that runs multiple full passes and appends a `(/Last30Days)` title tag for clearer output differentiation.

Beyond these headline features, the release touches several workflow improvements. Evidence lines can now attribute top comments using both `u/` and `@` handles, inline markdown links are rendered in narrative citations, and the Codex module gains a configurable memory directory (contributed by @davemorin). A fix was also applied to Step 0.55's category-peer subreddit expansion logic, and the release introduces a canonical `SKILL.md` default alongside a quieter sub-run experience with a default setting of 2.

Why this matters

  • The official X API v2 OAuth2 backend replaces potentially fragile scraping with a sanctioned, stable interface, reducing risk of access disruption.
  • The new --competitors flag automates comparison fan-out that previously required manual configuration, lowering the barrier for competitive research workflows.
  • The consolidation of five point releases (3.0.10–3.0.14) into a single stable tag simplifies upgrade paths for existing users.
