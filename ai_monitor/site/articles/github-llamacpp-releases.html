llama.cpp Ships SSM_SCAN Support for WebGPU Backend in Release b8926

The llama.cpp project released version b8926 on April 25, 2026, adding State Space Model scan support to its WebGPU backend alongside broader platform and CUDA compatibility updates.

GitHub llama.cpp Releases 2026-04-25 08:14 UTC Key: github-llamacpp-releases::identity::GitHub llama.cpp Releases Confidence: moderate Mode: claude

Article body

The llama.cpp project, maintained under ggml-org, has published release b8926, introducing support for the SSM_SCAN operation within its WebGPU backend. This addition, tracked as PR #22327, also disables error checking for the set_rows function in that context, and includes binding fixes and updated operator support to improve compatibility. The release further removes blocking behaviour in graph_compute and introduces set_rows validation checks in other compute paths. Beyond the WebGPU work, b8926 ships expanded CUDA support on Windows, including pre-built DLLs for CUDA 12.4 and CUDA 13.1 (13.1 DLLs specifically), alongside ongoing Vulkan support. Linux builds continue to cover a wide range of targets — CPU variants across x64, arm64, and s390x architectures, Vulkan builds for x64 and arm64, ROCm 7.2, OpenVINO, and SYCL configurations in both FP32 and FP16 precisions. macOS/iOS builds are maintained for Apple Silicon (including KleidiAI-enabled builds) and Intel x64, while Android and Windows ARM64 CPU builds remain part of the release matrix. The release was tagged on GitHub at 08:14 UTC on April 25, 2026, reflecting a continued rapid iteration cadence for the open-source inference engine, which is widely used for running large language models in resource-constrained environments.

Why this matters

  • SSM_SCAN support in WebGPU enables efficient state space model inference in browser and cross-platform GPU environments without relying on proprietary CUDA or Metal backends.
  • The expanded CUDA 13.1 DLLs on Windows give developers access to the latest NVIDIA toolkit features and performance improvements directly through llama.cpp binaries.
  • Continued broad platform coverage — from s390x Linux to Android arm64 — ensures llama.cpp remains one of the most portable LLM inference runtimes available.
