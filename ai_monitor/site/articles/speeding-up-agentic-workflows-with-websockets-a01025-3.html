Speeding Up Agentic Workflows with WebSockets in the Responses API

OpenAI has added WebSocket support to its Responses API, cutting end-to-end latency for agentic loops by roughly 40 percent.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When Codex fixes a bug in a codebase, it fires dozens of sequential requests to the Responses API—determining the model's next action, running a local tool, sending the result back, and repeating. As inference speeds have climbed, that cumulative API overhead became the dominant bottleneck. OpenAI's engineering team set out to close that gap and, after a two-month sprint, shipped WebSocket mode to production.

The core problem was structural: each follow-up request reprocessed the full conversation history, rebuilt context, and paid the cost of a new HTTP handshake. The fix was a persistent connection that caches reusable state in memory for the lifetime of the session. Using WebSockets rather than gRPC bidirectional streaming kept the existing `response.create` input and output shapes intact, so developers could adopt the mode without rewriting their integrations. When a follow-up request includes `previous_response_id`, the server fetches cached state—including the prior response object, tool definitions, and previously rendered tokens—instead of recomputing everything from scratch.

The results were immediate. Alpha testers reported up to 40% improvements in their agentic workflows. Codex ramped the majority of its Responses API traffic onto WebSocket mode within days of launch, and GPT-5.3-Codex-Spark hit the team's 1,000 tokens-per-second target, with bursts reaching 4,000 TPS. Third-party integrations followed: Vercel's AI SDK saw latency drop by roughly 40%, Cline's multi-file workflows ran 39% faster, and OpenAI models in Cursor became up to 30% faster.

Why this matters

  • Faster inference only benefits users if the surrounding infrastructure can keep up; persistent WebSocket connections eliminate the API overhead that was throttling agentic performance.
  • Developers can adopt WebSocket mode without changing their existing Responses API input and output shapes, lowering the barrier to meaningful latency gains.
  • The speed gains are already materializing across the ecosystem: Codex, Vercel, Cline, and Cursor have all shipped measurable improvements to end users.
