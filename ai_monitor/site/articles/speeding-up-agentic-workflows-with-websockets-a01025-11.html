OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has enabled WebSocket mode on its Responses API, cutting end-to-end latency in agentic workflows by up to 40 percent by replacing a series of independent HTTP requests with a single persistent connection.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

As inference on GPT-5.3-Codex-Spark pushed past 1,000 tokens per second on Cerebras hardware, the cumulative overhead of repeated synchronous API calls became the new bottleneck in agentic loops involving dozens of back-and-forth requests per task.

The solution was a persistent WebSocket connection that keeps reusable conversation state cached in memory for the session lifetime. Clients send only new information rather than full conversation history, and the server reuses prior response state, tool definitions, and routing logic across requests. Critically, OpenAI preserved the familiar response.create API shape and a previous_response_id parameter, so developers can enable WebSocket mode without rewriting existing integrations.

After a two-month internal sprint and an alpha with coding agent startups, WebSocket mode shipped in production with Codex moving the majority of its Responses API traffic to the new mode within days. External adopters have reported significant gains: Vercel's AI SDK saw latency drop by up to 40 percent, Cline's multi-file workflows ran 39 percent faster, and OpenAI models in Cursor became up to 30 percent faster.

Why this matters

  • As LLM inference gets faster on specialized hardware, the surrounding API infrastructure becomes the new limiting factor for agentic applications—and this WebSocket approach directly addresses that shift.
  • Developers can adopt persistent connections without changing their API input and output shapes, making the upgrade path comparatively low-friction.
  • Early third-party integrations in Vercel, Cline, and Cursor show the performance gains are translating to real end-user experience improvements across the broader AI coding tooling ecosystem.
