OpenAI Ships Codex 0.125.0 with Unix Sockets, Plugin Marketplace Upgrades, and Revised Permission Profiles

OpenAI has published Codex version 0.125.0, delivering app-server Unix socket transport, remote plugin installation, and improved permission profile durability across TUI sessions.

GitHub Codex Releases 2026-04-24 23:21 UTC Key: github-codex-releases::merge::releasegithub-codex-releaseshttpsgithubcomopenaicodexreleasesatom Confidence: moderate Mode: claude

Article body

OpenAI released Codex version 0.125.0 on April 24, 2026, marking the latest stable update to its AI-powered coding agent. The release ships six major new features, including significant enhancements to app-server integrations — now supporting Unix socket transport, pagination-friendly resume and fork operations, sticky environments, and remote thread configuration plumbing.

The update also introduces remote plugin installation and marketplace upgrades for app-server plugin management, and rolls out a more robust permission profile system that now persists across TUI sessions, user turns, MCP sandbox states, shell escalation events, and app-server API calls. For developers using AWS Bedrock, model providers can now own model discovery with Bedrock account state exposed directly to app clients. The `codex exec --json` command has been updated to report reasoning-token usage for programmatic consumers.

Several bugs were also addressed in this release: interrupting `/review` and exiting the TUI no longer leaves the interface wedged on delegate startup or unsubscribe, and exec-server no longer drops buffered output after process exit, now correctly waiting for stream closure. A new `0.126.0-alpha.1` pre-release tag was also pushed on the same day, signaling that development is already moving forward.

Why this matters

  • Unix socket transport and sticky environments give enterprise teams more flexible, secure deployment options for Codex in server-side workflows.
  • The new permission profile persistence means developers no longer need to re-authorize actions repeatedly across sessions, reducing friction in long coding sessions.
  • AWS Bedrock model discovery directly in the Codex app client simplifies multi-cloud AI setups, making it easier to provision and switch models without manual configuration.
