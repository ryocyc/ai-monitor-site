llama.cpp b8908 fixes heap-buffer-overflow in server's n_discard handling

Release b8908 patches a heap-buffer-overflow vulnerability in llama.cpp's server component that could be triggered by sending a negative n_discard value in client JSON.

GitHub llama.cpp Releases 2026-04-23 23:18 UTC Key: github-llamacpp-releases::identity::b8908 Confidence: strong Mode: claude

Article body

The ggml-org/llama.cpp project issued release b8908 on April 23 to address a heap-buffer-overflow in the server's update_slots() context-shift loop. The flaw stems from a negative n_discard value accepted from client JSON, which causes memory corruption. The fix clamps n_discard to a non-negative value at the JSON parse boundary, with n_discard=0 already triggering the existing auto-discard behaviour (n_left/2). The vulnerability is classified as CWE-787 (out-of-bounds write) with a CVSS score of 8.8, and is tracked as CVE-2026-21869 (GHSA-8947-pfff-2f3c).

Beyond the two core code fixes, the release ships pre-built binaries for a wide range of platforms: macOS Apple Silicon (arm64, with and without KleidiAI), macOS Intel (x64), iOS XCFramework, multiple Linux builds across Ubuntu and openEuler architectures (x86, arm64, s390x) with CPU, Vulkan, ROCm 7.2, OpenVINO, and SYCL backends, Android arm64, and Windows x64/arm64 with CPU, CUDA 12/13, Vulkan, SYCL, and HIP support.

The release was co-authored by Georgi Gerganov and signed with a GitHub verified GPG key. Assets failed to load in the source page capture, so the full binary listing and any additional notes in the release body could not be confirmed.

Why this matters

  • CVSS 8.8 is a high-severity score; heap-buffer-overflow can potentially be exploited for memory corruption or information disclosure in server deployments.
  • The flaw is trivially reachable over the network — any client that sends crafted JSON to a llama.cpp server can trigger it without authentication mentioned in the release notes.
  • Binary assets cover 20+ platform/architecture combinations, indicating this is a widely-used server component that users likely run as an API endpoint.
