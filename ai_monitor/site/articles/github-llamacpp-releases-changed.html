llama.cpp release adds OpenCL acceleration for iq4_nl quantized models

The llama.cpp project shipped OpenCL support for the iq4_nl quantization format in release b8935, extending hardware-accelerated inference for this memory-efficient scheme beyond CUDA-dependent GPUs.

Article body

The llama.cpp project published release b8935 on April 26, 2026, adding three OpenCL-backed changes for the iq4_nl quantization format via pull request #22272: general iq4_nl support on OpenCL, GEMM/GEMV kernels optimized for Adreno GPUs, and a memory-efficiency improvement that packs two lookup-table entries into a single unsigned integer.

iq4_nl is a 4-bit integer quantization scheme within the GGUF model format designed to reduce memory footprint and speed up inference on compatible hardware while preserving model quality. The llama.cpp project is a C/C++ inference engine that runs GGUF-format large language models locally across a wide range of hardware backends including CPU, CUDA, Vulkan, HIP/ROCm, SYCL, OpenCL, and OpenVINO — covering macOS, Linux, Windows, Android, and iOS.

These changes let developers and teams run iq4_nl-quantized models on any OpenCL-capable hardware — including older AMD GPUs, integrated graphics, and mobile SoCs — without requiring CUDA. Inference engineers and application developers running local models at scale can expect lower memory costs on compatible hardware, and the same llama.cpp binary now handles multiple accelerator backends for this quantization type.

Why this matters

  • iq4_nl is a practical middle ground between model quality and memory efficiency; extending it to OpenCL hardware opens inference to a broader set of devices beyond CUDA-dependent GPUs.
  • Adreno-optimized GEMM/GEMV kernels in OpenCL can meaningfully improve throughput on Qualcomm-based hardware, including Android devices and Windows arm64 PCs.
  • The packed LUT optimization reduces memory overhead per quantized layer, which matters most for larger models or memory-constrained deployment scenarios.
