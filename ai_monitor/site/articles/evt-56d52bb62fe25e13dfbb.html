n8n 2.17.8 Patches Credit Prompt Bug and Refreshes Editor Wallet UI

The latest n8n release eliminates a conflicting "Get 100 free OpenAI API credits" prompt for users running aiGateway under a valid license, and fixes the editor's wallet balance display so it refreshes on demand.

Article body

n8n has published version 2.17.8, addressing two editor-level bugs in its workflow automation platform. The first fix prevents a "Get 100 free OpenAI API credits" prompt from appearing when aiGateway is enabled under an active license, removing an unnecessary and potentially confusing upsell flow for paying users. The second fix ensures the wallet balance in the editor refreshes immediately when a user clicks the usage refresh action.

Both changes are scoped to the editor component and tied to aiGateway, n8n's built-in routing layer for directing AI API calls through a unified gateway. The aiGateway feature is designed to let n8n users manage API credit allocation and routing across multiple AI providers from a single interface. The release notes reference two pull requests—#29058 for the credit prompt fix and #29081 for the wallet refresh—published on April 27, 2026.

For teams using n8n to orchestrate AI-powered workflows, the 2.17.8 patch removes an annoyance that could interrupt operator sessions, and the corrected wallet balance refresh means users can monitor actual API spend more reliably without needing to reload the full interface. Workflow authors who rely on aiGateway for credit pooling or provider routing should update to avoid the conflicting UI state.

Why this matters

  • License-paying n8n users running aiGateway were seeing a promotional credit prompt they couldn't act on, creating a confusing UX that has now been removed.
  • Accurate wallet balance visibility is essential for teams managing AI API spend at scale; the previous bug meant stale data after a usage refresh click.
  • The patch confirms ongoing investment in aiGateway's editor experience, signaling that n8n's AI gateway feature is a maintained and actively developed part of the platform.
