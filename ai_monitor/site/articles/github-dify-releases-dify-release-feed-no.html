Dify v1.13.3 Patches Workflow Runtime, Streaming, and Knowledge Retrieval

The Dify team has shipped v1.13.3, a patch release targeting stability across workflow execution, real-time streaming, and knowledge-base retrieval.

GitHub Dify Releases 2026-04-22 12:43 UTC Key: github-dify-releases::identity::github-dify-releases-dify-release-feed-no Confidence: strong Mode: claude

Article body

Dify v1.13.3 landed on March 27, 2026, arriving 718 commits after the v1.13.2 tag. The release is signed with the committer's verified GitHub signature, adding variable-reference support for model parameters in LLM, Question Classifier, and Variable Extractor nodes in the workflow builder.

On the bug-fix side, the team patched StreamsBroadcastChannel replay and concurrency issues to stabilize frontend/backend event delivery, restored correct prompt message transformation logic, and fixed handling where max_retries was set to zero in executor-driven HTTP Request nodes. The workflow editor also no longer lets pasted nodes carry stale Loop/Iteration metadata, and HumanInput nodes are blocked from landing in invalid containers.

Knowledge retrieval received several targeted corrections: citation metadata in web responses is now preserved, crashes triggered by missing dataset icon metadata are gone, hit-count query filtering works as expected, and indexed document chunk previews are restored in the knowledge base UI. The release notes caution that Python and Node.js paths for Sandbox are not auto-migrated from existing configuration files, so teams upgrading manually should update those paths to match the new defaults.

Why this matters

  • Workflow nodes now accept variable references for model parameters, enabling more dynamic LLM, classifier, and extraction pipelines.
  • Streaming event delivery is now reliable under replay and concurrent load, fixing a pain point for real-time frontend integrations.
  • Missing dataset metadata and corrupted chunk previews are resolved, preventing crashes that previously disrupted knowledge base queries.
