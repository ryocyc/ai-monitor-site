Vercel Skills v1.5.1 Skips LFS Smudge to Reduce Bandwidth During Git Clone

A targeted fix in Vercel Skills v1.5.1 stops Git Large File Storage from smudging files during clone, cutting unnecessary bandwidth usage for developers.

Article body

Version 1.5.1 of the Vercel Skills open-source project addresses a git operation inefficiency by skipping the LFS smudge step during repository clones. The change prevents Git LFS from downloading tracked file content prematurely, which had been causing wasteful bandwidth usage, particularly noticeable on large repositories with extensive LFS assets.

The fix was contributed by @jrusso1020 via pull request #952 and shipped as a single commit to the main branch. The release is listed as the latest version in the project's GitHub repository, which has accumulated roughly 15,800 stars and over 1,300 forks.
