Hugging Face Adds Auto-Generated /agents.md Endpoints to All Gradio Spaces

Every Gradio Space on Hugging Face now automatically exposes a machine-readable /agents.md endpoint, allowing AI coding agents to discover and interact with spaces without manual configuration.

Hugging Face Changelog 2026-04-25 04:52 UTC Key: hugging-face-changelog::merge::docshugging-face-changelogchangelog-keep-track-of-latest-changes-on-the-hugging-face-hub-apr-17-26-upvote-125-120-apr-17-26-spaces-agents-md-for-your-coding-age-changelog-keep-track-of-latest-changes-on-the Confidence: strong Mode: claude

Article body

Hugging Face has quietly rolled out a major usability upgrade for its Gradio Spaces platform: every Space now automatically serves an /agents.md endpoint. The endpoint provides a structured, machine-readable description of what the Space does and how it can be called, eliminating the need for developers to hand-write agent-integration documentation for each Space.

The practical effect is straightforward: AI coding agents such as Claude Code, OpenAI Codex, or Pi can now point themselves at a Gradio Space and immediately understand how to use it, without any prior setup or human-authored API docs. This closes a long-standing friction point in the AI tooling ecosystem, where agents routinely struggled to interact with third-party or community-hosted tools.

Also announced on the changelog is the launch of Kernels on the Hugging Face Kernels Hub. Kernels are precompiled, hardware- and PyTorch-version-specific compute kernels that are ready for torch.compile, delivering between 1.7× and 2.5× speed-ups over baseline PyTorch performance on compatible hardware.

Why this matters

  • AI agents can now autonomously discover and use thousands of Gradio Spaces without human intervention, dramatically lowering the barrier to building agentic workflows on the Hugging Face ecosystem.
  • Developers hosting Gradio Spaces gain built-in agent compatibility at no additional cost, since the /agents.md endpoint is generated automatically rather than requiring manual documentation work.
  • The separate Kernels Hub launch signals Hugging Face's intent to expand beyond model hosting into the broader ML tooling layer, giving researchers and engineers a practical performance optimization path via precompiled kernels.

Source note

  • Both features were announced on the official Hugging Face Changelog page (huggingface.co/changelog), with the /agents.md update posted on April 17, 2026 (125 upvotes) and the Kernels launch on April 15, 2026 (146 upvotes). The Kernels Hub is accessible at the Hugging Face Kernels Hub URL referenced in the announcement. Documentation links for /agents.md are available on the same changelog page.
