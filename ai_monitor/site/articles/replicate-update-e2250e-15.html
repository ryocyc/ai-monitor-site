Replicate Publishes Agent Skills to Guide AI Coding Assistants on Model Selection and Prompting

Replicate has started publishing "agent skills" — markdown instruction files that give AI coding assistants structured expert knowledge about working with its models, APIs, and prompting techniques.

Replicate Changelog 2026-04-22 23:49 UTC Key: replicate-changelog::identity::replicate-update Confidence: strong Mode: claude

Article body

The skills, announced on April 21, 2026, are installed via a single command: `npx skills add replicate/skills`. Once added, they configure themselves for compatible coding assistants automatically. The files are designed around the open Agent Skills spec, and Replicate confirms compatibility with Claude Code, OpenCode, and OpenAI Codex out of the box.

The content covers three broad areas: discovering and comparing models, executing predictions through the API, and writing effective prompts for image and video generation models specifically. Replicate draws a clear distinction between these skills and its existing MCP server — the latter provides API tools, while the skills layer supplies the judgment call of when and how to use those tools well, including guidance on model tradeoffs.

The skills and the MCP server are framed as complementary rather than redundant. For developers who rely on AI coding assistants to interact with Replicate programmatically, the skills aim to reduce the amount of trial-and-error around model selection and prompt construction.

Why this matters

  • Agent skills provide a standardized way for coding assistants to reason about Replicate's model catalog without relying on out-of-date documentation or guesswork.
  • The distinction between MCP (tools) and skills (knowledge) gives developers a clearer mental model for integrating Replicate into AI-assisted workflows.
  • The open Agent Skills spec means the same instruction format can be adopted by other platforms, potentially making multi-provider agentic workflows more consistent.
