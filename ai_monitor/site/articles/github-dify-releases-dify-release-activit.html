Dify v1.13.3 Patches Streaming Reliability and Workflow Editor Bugs

Dify's latest patch release, v1.13.3, adds variable-reference support for model parameters across three node types while resolving replay and concurrency issues in its streaming layer.

GitHub Dify Releases 2026-04-22 15:01 UTC Key: github-dify-releases::identity::github-dify-releases-dify-release-activit Confidence: strong Mode: claude

Article body

The v1.13.3 release introduces variable-reference support for model parameters in LLM, Question Classifier, and Variable Extractor nodes, allowing more dynamic workflow configurations. This change lets users wire model-level settings to upstream variables rather than hardcoding them per node.

The patch fixes a pair of StreamsBroadcastChannel issues that caused replay and concurrency problems in frontend/backend event delivery, aiming to keep real-time streaming stable. It also corrects pasted nodes incorrectly retaining Loop/Iteration metadata, and prevents HumanInput nodes from being pasted into invalid containers.

Knowledge retrieval sees multiple fixes: citation metadata is now preserved in web responses, crashes when dataset icon metadata is missing have been addressed, hit-count query filtering is corrected, and indexed document chunk previews are restored. Runtime execution improvements restore prompt message transformation logic and fix max_retries=0 handling for executor-driven HTTP requests.

Why this matters

  • Variable-reference support for model nodes gives workflow authors more flexibility without duplicating node configurations.
  • Streaming reliability fixes directly affect users relying on real-time event delivery between Dify's frontend and backend.
  • Knowledge retrieval fixes target usability issues that could cause crashes or degrade citation quality in production knowledge bases.
