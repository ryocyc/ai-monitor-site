OpenAI Adds WebSockets to Responses API, Speeding Up Agentic Workflows by 40%

OpenAI has added WebSocket support to its Responses API, allowing developers to maintain a persistent connection that dramatically reduces the overhead of agentic AI loops.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When an AI agent like Codex fixes a bug, it fires off dozens of sequential requests to the Responses API — each one carrying the full conversation history, reprocessing context that mostly hasn't changed. As inference speeds have climbed, that repeated API overhead became a growing bottleneck. OpenAI's solution: a persistent WebSocket connection that keeps state cached in memory between calls, so only genuinely new information needs to be validated and processed each round.

The key technical shift is that the server now maintains a connection-scoped, in-memory cache of prior response state — including rendered tokens, tool definitions, and prior input/output items — which is reused across follow-up requests using a `previous_response_id`. Safety classifiers and request validators also process only new input rather than re-scanning the full conversation history. Non-blocking post-inference work like billing is overlapped with subsequent requests, squeezing more efficiency out of the pipeline.

Early results from alpha testers were strong enough to ship quickly. Codex migrated the majority of its Responses API traffic onto WebSocket mode and hit burst throughput of up to 4,000 tokens per second on GPT-5.3-Codex-Spark. Independent integrations confirmed the gains: Vercel's AI SDK saw up to 40% latency reduction, Cline's multi-file workflows ran 39% faster, and Cursor's OpenAI models improved by up to 30%.

Why this matters

  • As inference gets faster, API overhead becomes the dominant latency factor in agentic loops — WebSockets directly address that shift.
  • Developers can enable WebSocket mode without changing their existing `response.create` input and output shapes, making adoption straightforward.
  • Faster agent loops mean coding assistants and other agentic tools feel more responsive to end users in real production traffic.
