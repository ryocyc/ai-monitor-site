Continue IDE Extension v1.3.38 Fixes config.yaml Initialization and Adds Workspace Filtering

The latest release of the Continue VS Code extension addresses a long-standing issue where config.yaml was not properly created on first access, while also adding a new session history filter scoped to the current workspace.

Article body

Continue has released version 1.3.38 of its VS Code extension (with corresponding releases v1.2.22 for older VS Code and v1.0.67 for JetBrains IDEs), fixing a bug where config.yaml was not properly initialized or populated when first accessed by a user. The fix, submitted by contributor @RomneyDa in pull request #11915, ensures the configuration file is created and populated automatically, removing a friction point for new users setting up the tool.

The release also introduces session history filtering by workspace directory, allowing developers to view only the conversation history relevant to their current project. This feature was contributed by @mvanhorn in pull request #11813. Separately, the release adds support for a `.continue/configs` directory, enabling more flexible configuration management for users managing multiple project setups.

The JetBrains-specific release (v1.0.67) includes an additional fix that redirects Winston logger output to stderr to prevent IPC stream corruption and resolves related binary test failures. All three versions carry routine dependency updates across multiple internal packages including handlebars, happy-dom, brace-expansion, and path-to-regexp, and resolve several compatibility issues across the core, CLI, and SDK packages.

Why this matters

  • A misconfigured or missing config.yaml could silently block first-time users from getting started; the fix removes that silent failure.
  • Workspace-scoped session history makes it easier to find relevant past conversations without scrolling through unrelated project chats.
  • The `.continue/configs` support enables per-project configuration overrides, a common need for teams sharing dotfile setups.
