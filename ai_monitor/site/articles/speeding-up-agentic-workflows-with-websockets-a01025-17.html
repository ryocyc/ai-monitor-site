OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, cutting end-to-end latency for agentic workflows by up to 40% by replacing sequential HTTP calls with a persistent, stateful connection.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When you ask Codex to fix a bug, it scans your codebase, reads files, makes edits, and runs tests — a process that previously required dozens of synchronous Responses API calls. As inference speeds have climbed past 1,000 tokens per second, that cumulative API overhead became the new bottleneck. OpenAI's solution is a WebSocket mode that maintains a persistent connection and caches conversation state in memory, so follow-up requests only send new information rather than the full history each time.

The design keeps the familiar `response.create` API shape and uses a `previous_response_id` to resume from a server-side cache containing the prior response object, input/output items, tool definitions, and rendered tokens. Safety classifiers and request validators process only new input rather than re-evaluating the entire conversation history, and non-blocking postinference work like billing overlaps with subsequent requests.

The feature landed after a two-month sprint and an alpha with key coding-agent startups. Codex rapidly moved the majority of its Responses API traffic onto WebSocket mode, and the impact has spread across the developer ecosystem: Vercel's AI SDK saw up to 40% latency reduction, Cline's multi-file workflows are 39% faster, and OpenAI models in Cursor are up to 30% faster.

Why this matters

  • Agentic AI tools like Codex depend on dozens of back-and-forth API calls per task — WebSockets turn that loop into a single persistent session, eliminating redundant work.
  • The feature enables GPT-5.3-Codex-Spark to hit its 1,000 tokens-per-second target in production traffic, with bursts reaching 4,000 TPS, without users waiting on API overhead.
  • OpenAI says WebSocket mode is one of the most significant new capabilities in the Responses API since its March 2025 launch, and it applies to all major recent models including GPT-5.3-Codex, GPT-5.4, and beyond.
