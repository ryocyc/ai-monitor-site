llama.cpp b8893 Flips HIP Graphs to Default-On for AMD GPUs

The llama.cpp project has turned on HIP graph execution by default for AMD GPUs, citing positive performance results from ROCm improvements and graph-construction refinements.

GitHub llama.cpp Releases 2026-04-23 02:36 UTC Key: github-llamacpp-releases::identity::github-llamacpp-releases-llamacpp-release-fee Confidence: moderate Mode: claude

Article body

Release b8893 of llama.cpp changes the default for GGML_HIP_GRAPHS to enabled on AMD hardware. The toggle was disabled in an earlier revision (#11362) because graph execution hurt performance at the time. The project says that ROCm itself and how the codebase builds and uses graphs have both improved enough to flip that default.

The release ships prebuilt binaries across a broad set of targets: CUDA 13.1 and 12.4 on Windows, Vulkan on Ubuntu and Windows, ROCm 7.2 on Ubuntu, SYCL and HIP on Windows, OpenVINO on Linux, KleidiAI-enabled builds for Apple Silicon, and iOS XCFramework packages. OpenEuler builds with 310p and 910b (ACL Graph) variants are also listed for both x86 and arm64.

The commit was signed and verified on GitHub, with the release published on 23 April 2026 at 02:36 UTC. Three community members reacted positively to the announcement.

Why this matters

  • AMD GPU users on ROCm can now run llama.cpp with HIP graph execution active out of the box, potentially improving throughput without extra configuration.
  • Prebuilt binaries for CUDA 13.1 and Vulkan on Windows lower the barrier to running llama.cpp on those platforms.
  • The KleidiAI-enabled Apple Silicon build gives M-series Mac users an additional optimised path for inference.
