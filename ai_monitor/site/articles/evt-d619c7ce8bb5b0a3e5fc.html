n8n Ships Version 1.123.38 With CLI Credential Import Filters and Task Runner Fixes

The open-source workflow automation platform patches two stability issues in its core execution engine while adding filtering flags to its CLI credential import and export tools.

Article body

n8n released version 1.123.38 on April 29, 2026, closing two reliability bugs in the platform's core and task runner alongside new command-line filtering capabilities. The release is tagged as v1 in the project's release feed.

The bug fixes address race conditions in n8n's execution engine: code node executions could hang when an idle timer overlapped with task acceptance, and the task runner itself could stall when a connection attempt failed. Both issues were tracked via pull requests #29393 and #29441 respectively. In parallel, the CLI tooling received new flags ã `--include` and `--exclude` for the `import:credentials` command, and a `--projectId` filter for `export:workflow` and `export:credentials` ã giving operators more granular control when managing workflows and credentials at scale from the terminal.

For teams running n8n in production, the task runner and code node fixes eliminate execution deadlocks that could silently stall automations. The new CLI filters are practical for DevOps engineers automating credential rotation or workflow migrations across projects without pulling in the entire credential store.

Why this matters

  • Task runner hangs could cause long-running automations to stall indefinitely in production environments, creating silent failures in workflows that operators might not notice until downstream systems miss expected outputs.
  • The new CLI `--include` and `--exclude` flags for credential imports, along with the `--projectId` filter on exports, give MLOps and DevOps teams a surgical way to manage credentials in multi-project n8n deployments without a full re-import.
  • The v1 release feed tag signals that the n8n team is treating this as a meaningful milestone point, which developers building automation pipelines and integrations may want to reference for version planning.
