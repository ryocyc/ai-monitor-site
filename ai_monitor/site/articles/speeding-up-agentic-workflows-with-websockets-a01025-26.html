OpenAI's Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, enabling persistent connections that can cut end-to-end agent loop latency by up to 40 percent for coding tasks.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

The change targets the overhead that accumulates when agents like Codex make dozens of back-and-forth API requests in a single task. Previously, each follow-up request re-sent the full conversation history, re-tokenized context, and re-validated safety—work that repeated even when most of the state hadn't changed. With WebSocket mode, the server maintains a connection-scoped, in-memory cache of previous response state, so `response.create` calls with a `previous_response_id` fetch cached context rather than rebuilding it from scratch.

The performance gap became acute as inference speeds climbed. OpenAI's earlier flagship models ran at roughly 65 tokens per second, making API overhead easy to hide. The newer GPT-5.3-Codex-Spark targets over 1,000 tokens per second, powered by specialized Cerebras hardware, which means API processing time became the new bottleneck. A November 2025 performance sprint delivered a 45 percent improvement in time-to-first-token, but still left overhead too high relative to inference speed.

After a two-month alpha with key coding agent startups, WebSocket mode shipped broadly. Codex moved most of its Responses API traffic onto WebSockets and hit bursts of 4,000 tokens per second in production. External adopters reported gains too: Vercel's AI SDK saw latency drop by up to 40 percent, Cline's multi-file workflows ran 39 percent faster, and Cursor reported up to 30 percent improvements with OpenAI models.

Why this matters

  • API overhead was the new bottleneck as inference speeds crossed 1,000 tokens per second, and persistent WebSocket connections eliminate the repeated overhead of re-processing full conversation history per request.
  • Developers can adopt WebSocket mode without changing their existing `response.create` call shapes—they only add `previous_response_id` and connect via WebSocket.
  • Third-party integrations in tools like Vercel AI SDK, Cline, and Cursor are already reporting 30–40 percent latency improvements in production coding workflows.
