OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI's Responses API now supports WebSockets, cutting end-to-end latency for agentic AI workflows by 40% while enabling a new coding model to run at over 1,000 tokens per second.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

OpenAI has added WebSocket support to its Responses API, enabling developers to maintain a persistent connection that caches conversation state in memory rather than sending the full history with every request. The change directly addresses a bottleneck that emerged as inference speeds outpaced the overhead of repeated HTTP handshakes. Where earlier flagship models like GPT-5 ran at roughly 65 tokens per second, OpenAI's newer GPT-5.3-Codex-Spark achieves over 1,000 tokens per second on specialized Cerebras hardware, making API-level latency the new constraint.

The engineering team approached this through several optimizations: caching rendered tokens and model configuration in memory, eliminating calls to intermediate services by routing directly to the inference service, and improving the safety stack so classifiers process only new input rather than replaying the full conversation history. The critical shift was switching from treating each Codex agent request as independent to treating an entire agentic rollout as a single long-running response connected via WebSocket. With the connection scoped in-memory cache in place, multiple requests can overlap across validation, preinference, sampling, and postinference stages rather than running sequentially.

After a two-month sprint and an alpha with key coding agent startups, the results were immediate. Codex ramped the majority of its Responses API traffic to WebSocket mode within days of launch and hit bursts of 4,000 tokens per second in production traffic. Third-party integrations showed broad impact: Vercel's AI SDK saw latency drop by up to 40%, Cline's multi-file workflows ran 39% faster, and OpenAI models in Cursor became up to 30% faster. OpenAI called WebSocket mode one of the most significant new capabilities in the Responses API since its launch in March 2025.

Why this matters

  • AI inference is getting dramatically faster, but the surrounding infrastructure must keep pace to deliver those gains to users.
  • WebSocket mode lets developers drop in persistent connections without changing their existing API input and output shapes.
  • Major developer tooling—Codex, Vercel, Cline, Cursor—already reported substantial latency gains, making this immediately usable.
