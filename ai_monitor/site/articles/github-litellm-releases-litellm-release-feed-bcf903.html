LiteLLM v1.83.11-nightly Lands Bedrock Mantle Endpoint and Proxy-Level Prompt Compression

The latest nightly build of the popular LLM proxy library adds a new Bedrock Mantle endpoint, introduces prompt compression directly into the proxy layer, and patches a critical 503 loop that could occur when the database is unreachable.

Article body

BerriAI released LiteLLM v1.83.11-nightly on April 22, 2026, bringing a cluster of proxy-focused improvements alongside a new AWS Bedrock endpoint variant. The standout addition is a `--reload` flag for uvicorn that enables hot reload during development, along with native prompt compression baked into the proxy itself. The release also marks the debut of an opt-in v2 migration resolver for the proxy tier.

On the cloud provider side, Scaleway gains first-class audio support, and a new Bedrock Mantle endpoint (tagged as a Claude Mythos Preview) is now available. A related fix tightens Bedrock invoke body allowlisting, filtering all `anthropic-beta` values for safety.

Several reliability fixes round out the release. A notable regression — a 503 loop triggered by `/health/readiness` when the database is unreachable — has been patched. OAuth endpoint access controls for the MCP broker were hardened, and both user and org budget spend checks were aligned using an atomic counter pattern to prevent race conditions.

Why this matters

  • Prompt compression in the proxy layer can reduce token costs and latency without application-level changes.
  • The Bedrock Mantle endpoint opens access to a new Claude model variant via AWS infrastructure.
  • The 503 health-loop fix prevents a cascading failure scenario in proxy deployments with transient database connectivity.
