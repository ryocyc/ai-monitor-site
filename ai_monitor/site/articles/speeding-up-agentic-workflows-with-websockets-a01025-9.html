OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, cutting end-to-end latency for agentic AI workflows by up to 40 percent by replacing a series of synchronous HTTP calls with a persistent connection that reuses cached conversation...

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When you ask Codex to fix a bug, it scans your codebase, reads files, makes edits, and runs tests—often dozens of back-and-forth Requests API calls stacked together. As inference speeds have climbed dramatically, that overhead became the new bottleneck. For GPT-5.3-Codex-Spark, OpenAI's goal was north of 1,000 tokens per second, but the cumulative API overhead made that speed invisible to users. The fix was a structural rethink of how the client and API exchange data. Instead of opening a fresh HTTP connection and resending the entire conversation history with every follow-up request, WebSocket mode maintains a persistent connection that caches response state in memory across the lifetime of the session. This lets safety classifiers and request validators process only new input rather than re-evaluating the full history, skips redundant tokenization by appending to a cached rendered-token stream, and overlaps billing and other postinference work with subsequent requests.

Why this matters

  • Agentic workflows that rely on repeated API calls—like code completion, multi-file editing, and automated testing—can now run significantly faster without changes to the API input and output shapes developers already use.
  • GPT-5.3-Codex-Spark hit bursts of 4,000 tokens per second in production, demonstrating that the Responses API can keep pace with much faster specialized inference hardware, removing the API layer as a constraint.
  • Major developer tooling—Vercel AI SDK, Cline, and Cursor—have already integrated WebSocket mode, reporting latency improvements of 30–40% for end users.
