CrewAI 1.14.4a1 Lands Chat Resilience Fixes, Better Multimodal File Handling, and Security Patch

The CrewAI open-source multi-agent framework patched a failure mode in crew chat description helpers, improved how task-only agents handle multimodal file inputs, and upgraded its litellm dependency to close a server-side template injection vulnerability.

Article body

CrewAI released version 1.14.4a1 on 29 April 2026, a point release focused on bug fixes and documentation. The most notable change addresses a failure in crew chat description helpers triggered by certain LLM responses, a condition that could interrupt execution in multi-agent workflows relying on that hook. The release also resets messages and iteration counters between invocations in the executor, preventing carryover state that caused unpredictable behavior in long-running replays and tests.

The update also fixes how task-only agents bind to crews during execution, ensuring that multimodal input files reach the underlying LLM rather than being dropped silently. Custom trained-agents files are now correctly forwarded through both the replay and test commands in the CLI and honored at inference time in agents. Guardrail callable serialization was patched so that it serializes as null during JSON checkpointing, avoiding a class-inheritance error in that code path. On the security side, the litellm dependency was bumped to patch a server-side template injection (SSTI) vulnerability, while an unfixable pip CVE is now explicitly ignored. The agent_executor flag force_final_answer was renamed to avoid a self-referential router conflict.

Documentation additions cover three new integration guides: E2B Sandbox Tools, Daytona sandbox tools, and Vertex AI workload identity setup. A You.com integration page was also added. These additions signal that the CrewAI team is investing in concrete deployment pathways for agents built on its framework, particularly around sandboxed execution and cloud credential management.

Why this matters

  • Agent developers relying on crew chat description helpers can no longer assume stable behavior; the fix resolves a failure mode that could silently break multi-step workflows.
  • Multimodal file handling for task-only agents was a known gap; the fix ensures files reach the LLM as intended, removing a source of silent data loss.
  • The SSTI patch in litellm closes a code-injection vector that could affect any CrewAI deployment making LLM calls through that library, making the upgrade relevant even for teams not using the specific chat helper.
