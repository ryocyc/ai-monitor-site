n8n Brings AI Agents Into Microsoft 365 Apps via Agent 365 GA Integration

With Microsoft Agent 365 now generally available, n8n users can deploy workflow-powered AI agents as team members inside Outlook, Teams, and other Microsoft 365 surfaces, letting colleagues @mention them just like human collaborators.

Article body

n8n has announced that its users can now build and deploy AI agents that appear as full team members inside Microsoft 365 applications, powered by the general availability of Microsoft Agent 365. The integration allows n8n-built agents to be @mentioned directly within the Microsoft apps where work already happens — including Outlook and Teams — eliminating the need to switch context to a separate automation tool.

The feature taps into n8n's existing workflow automation capabilities, letting teams define agent behavior through n8n's visual workflow builder before surfacing those agents as conversational participants in Microsoft 365. According to the announcement, an agent built in n8n can receive a direct @mention from a colleague and respond within the same thread or inbox, performing tasks like querying knowledge bases, initiating follow-up workflows, or retrieving status updates without leaving the Microsoft ecosystem.

For developers and operations teams already using n8n for workflow orchestration, the integration offers a path to embed intelligent automation directly into employee-facing Microsoft tools without building custom connectors or maintaining separate bot infrastructure. The feature positions n8n as an agent-factory layer sitting between workflow logic and Microsoft 365 collaboration surfaces.

Why this matters

  • AI agents built with n8n can now live where workers already are — inside Outlook and Teams — rather than requiring them to open a separate automation tool.
  • The integration is powered by Microsoft Agent 365 GA, signaling that this is an officially supported and stable Microsoft surface, not a lab experiment.
  • For teams using Microsoft 365 as their primary collaboration stack, the feature could reduce friction in deploying custom AI automations without requiring specialized bot-development expertise.
