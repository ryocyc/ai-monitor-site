LiteLLM v1.83.12-nightly Adds Moonshot k2.6 and Cosign Docker Signing

This nightly build integrates the Moonshot/kimi-k2.6 model into LiteLLM's registry and begins signing all Docker images with Cosign for supply-chain verification.

GitHub LiteLLM Releases 2026-04-24 01:24 UTC Key: github-litellm-releases::identity::v18312-nightly Confidence: strong Mode: claude

Article body

The release adds the Moonshot/kimi-k2.6 model to LiteLLM's model registry, expanding the library's growing roster of supported providers. Two pull requests authored by Ishaan Berri (#26203, #26213) introduced the model and a related branch, while contributor @krrish-berri-2 contributed adaptive routing logic that can route requests across multiple model deployments dynamically.

A notable infrastructure addition is Cosign-based Docker image signing for all LiteLLM images published to ghcr.io. Verification can be performed using a pinned Git commit hash (recommended) or a release tag, both resolving to the same signing key committed in 0112e53. The release notes include the full Cosign verification commands for each method.

Other changes in the release include fixes for spend accuracy test flakiness and a Redis buffer data-loss path, a Prisma cache path restoration for Docker builds, migration of more CI jobs from CircleCI to GitHub Actions, Bedrock guardrail spend-log improvements in the proxy, wildcard routing fallback logic, a numeric UID fix for Kubernetes non-root runs, and a per-member spend tracking feature for team memberships.

Why this matters

  • Docker image signing with Cosign adds a verifiable supply-chain layer, letting users cryptographically confirm they are pulling genuine LiteLLM images rather than tampered ones.
  • The addition of Moonshot/kimi-k2.6 expands LiteLLM's multi-model orchestration to another frontier model provider, broadening supported use cases.
  • The adaptive routing feature enables LiteLLM to route requests across deployments intelligently, potentially improving reliability and cost efficiency for production deployments.
