OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to the Responses API, cutting end-to-end latency for agentic AI workflows by up to 40 percent by replacing synchronous HTTP request chains with a single persistent connection.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

The core problem was structural: agent loops like Codex's—scanning codebases, reading files, making edits, and running tests—required dozens of back-and-forth Responses API calls, each rebuilding conversation state from scratch. As inference speeds climbed toward 1,000 tokens per second on specialized Cerebras hardware, the cumulative API overhead became the new bottleneck, dwarfing the time spent on actual model inference. The solution was to rethink the transport layer entirely, keeping a persistent WebSocket connection so the server could cache and reuse prior response state—including rendered tokens, tool definitions, and conversation history—rather than reprocessing the full context on every follow-up request.

With this cached-state approach, the API could optimize several steps that previously repeated on each turn: safety classifiers and request validators now process only new input instead of re-evaluating the entire history, tokenization is skipped in favor of appending pre-rendered tokens, and non-blocking post-inference tasks like billing overlap with subsequent requests. The design preserves the familiar `response.create` shape and uses `previous_response_id` to tie each new request into the cached state on the server side, allowing developers to adopt WebSocket mode without rewriting their API integration.

After a two-month sprint, OpenAI launched the feature in alpha with key coding agent partners, who reported up to 40 percent improvements in agentic workflow latency. The results carried into production immediately: Codex moved the majority of its Responses API traffic onto WebSocket mode, Vercel integrated it into the AI SDK, Cline saw multi-file workflows run 39 percent faster, and Cursor reported up to 30 percent speed gains across OpenAI models.

Why this matters

  • As model inference accelerates toward thousands of tokens per second, API-layer overhead becomes the dominant latency factor—and WebSocket mode addresses that by eliminating redundant per-request processing.
  • Developers can adopt the performance gain without changing their API input/output shapes, lowering the barrier to integrating faster agentic workflows into existing tooling.
  • Broad third-party adoption by Codex, Vercel, Cline, and Cursor signals that WebSocket mode is already becoming a standard layer in the agentic development stack.
