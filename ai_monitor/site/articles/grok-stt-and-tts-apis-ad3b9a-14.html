Grok Speech to Text and Text to Speech APIs

xAI has released two standalone audio APIs — Grok Speech to Text and Grok Text to Speech — built on the same infrastructure powering Grok Voice in Tesla vehicles and Starlink support.

xAI Blog 2026-04-22 23:49 UTC Key: xai-blog::identity::grok-stt-and-tts-apis Confidence: strong Mode: claude

Article body

The Grok Speech to Text API handles both batch transcription and real-time streaming via REST and WebSocket endpoints. It includes word-level timestamps, speaker diarization, multichannel audio support, and Inverse Text Normalization that converts spoken numbers, dates, and currencies into structured output. The company benchmarks Grok STT at a 6.9% overall word error rate, comparing favourably against ElevenLabs (9.0%), Deepgram (11.0%), and AssemblyAI (12.9%) across phone calls, video, meetings, and telephony. The API supports 25 or more languages and allows seamless language switching mid-session.

The Grok Text to Speech API generates natural, expressive voices using simple inline speech tags such as `[laugh]`, `[sigh]`, and `[whisper]`, without requiring complex markup. Both TTS endpoints offer REST and WebSocket interfaces for long-form generation and real-time streaming.

Pricing is straightforward: STT costs $0.10 per hour for batch processing and $0.20 per hour for streaming, while TTS is priced at $4.20 per million characters. Full rate limits and documentation are available through the xAI API console.

Why this matters

  • xAI is entering a crowded speech API market with a benchmark claim that Grok STT outperforms all major competitors on entity recognition across business domains.
  • Inline speech tags give developers an unusually simple control mechanism for expressive TTS, potentially lowering the barrier for voice agent and interactive audio applications.
  • The same stack already runs Grok Voice in production environments — Tesla vehicles and Starlink customer support — which signals real-world robustness beyond benchmark numbers.
