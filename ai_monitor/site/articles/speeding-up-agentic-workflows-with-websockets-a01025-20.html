OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, cutting end-to-end latency for agentic workflows by roughly 40 percent by eliminating redundant API overhead across multi-turn agent loops.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When a coding agent like Codex fixes a bug, it fires dozens of back-and-forth requests to the Responses API—determining the model's next action, running a local tool, sending the result back, and repeating. As inference has grown faster, that cumulative API overhead became a bottleneck users could actually feel. OpenAI's engineering team set out to close that gap, and landed on WebSockets as the solution.

The core shift is from treating each request as independent to maintaining a persistent WebSocket connection with a server-scoped, in-memory cache of previous response state. When a follow-up request carries a `previous_response_id`, the server reuses that cached state—including prior input/output items, tool definitions, and already-rendered tokens—instead of reprocessing the full conversation history from scratch. Safety classifiers, request validators, and billing logic can all operate only on new input, and non-blocking postinference work overlaps with subsequent requests.

The results were immediate. After a two-month sprint, the alpha showed roughly 40% improvement in agentic rollout latency, and the production launch was taken up quickly by Codex, Vercel (AI SDK), Cline, and Cursor—with reported gains of 30–40% in real-world workflows. GPT-5.3-Codex-Spark hit OpenAI's 1,000 tokens per second target, with bursts reaching 4,000 TPS.

Why this matters

  • Agentic loops that once minutes of latency are now meaningfully faster, making AI coding tools more responsive in practice.
  • The familiar `response.create` API shape was preserved; developers drop in WebSocket mode without rewriting their integration.
  • The pattern—persistent connection plus incremental, stateful processing—points to how AI infrastructure must evolve as model inference continues to accelerate.
