OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, reducing end-to-end latency for agentic AI workflows by up to 40 percent by eliminating the overhead of repeated HTTP requests.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When developers run agents like Codex—where dozens of back-and-forth API calls pile up as the model scans code, makes edits, and runs tests—every millisecond of per-request overhead compounds into minutes of wait time. As inference speeds have climbed from roughly 65 tokens per second to over 1,000 tokens per second with GPT‑5.3‑Codex‑Spark, OpenAI found that the cumulative overhead of processing each HTTP round-trip had become the new bottleneck, dwarfing the GPU inference time it once dominated.

To solve this, OpenAI introduced a WebSocket transport mode for the Responses API that maintains a persistent connection and caches state—including prior input/output items, tool definitions, and previously rendered tokens—in memory across a session. Instead of rebuilding the full conversation from scratch on every call, clients send only new information, and the server reuses cached state when a `previous_response_id` is provided. Safety classifiers and request validators process only the incremental new input rather than re-evaluating the entire history, and non-blocking tasks like billing run in parallel with subsequent requests.

The feature was validated with an alpha program over roughly two months, after which Codex, Vercel, Cline, and Cursor all reported significant speedups. Codex shifted the majority of its Responses API traffic to WebSocket mode, and GPT‑5.3‑Codex‑Spark hit bursts up to 4,000 tokens per second in production. Developers can use the familiar `response.create` call shape without redesigning their integrations, since the message transport works underneath the existing API surface.

Why this matters

  • As AI models get faster at inference, the surrounding infrastructure—APIs, network hops, request overhead—increasingly becomes the limiting factor, and persistent WebSocket connections address that gap directly.
  • Developers using Codex, Vercel AI SDK, Cline, and Cursor are already reporting latency reductions of 30–40 percent in real-world agentic workflows, with no changes needed to their API input/output shapes.
  • The approach eliminates redundant work like re-tokenizing unchanged conversation history and re-running safety checks on previously processed content, compressing the entire agent loop into a tighter pipeline.
