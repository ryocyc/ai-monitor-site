LiteLLM v1.83.12-nightly Lands Moonshot/kimi-k2.6 and Cosign-Signed Docker Images

The latest nightly release of LiteLLM adds the Moonshot/kimi-k2.6 model to its registry, introduces adaptive routing, and verifies all Docker images via cosign.

GitHub LiteLLM Releases 2026-04-24 01:24 UTC Key: github-litellm-releases::identity::v18312-nightly Confidence: strong Mode: claude

Article body

LiteLLM's v1.83.12-nightly release adds the Moonshot/kimi-k2.6 model to its model registry, giving users access to that provider through LiteLLM's unified interface. The release also ships a new adaptive routing feature, which allows the library to dynamically direct requests across model deployments based on real-time conditions.

A notable infrastructure change is the addition of cosign-based signature verification for all LiteLLM Docker images hosted on ghcr.io. The signing key was introduced in commit 0112e53 and all images for this release carry that signature. Users can verify authenticity either by pinning to the commit hash or by using the release tag, both documented with the full cosign verify command.

The release also includes fixes and improvements spanning Docker caching behavior for pre-uv Prisma paths, CI migration from CircleCI to GitHub Actions, spend accuracy test stabilization, and a fix for wildcard ordering fallback to higher-order deployments. The GPG signature on the release commit was verified with GitHub's own key (ID: B5690EEEBB952194).

Why this matters

  • Cosign signatures on Docker images give operators a cryptographically verifiable chain of trust, reducing the risk of pulling tampered or unofficial images.
  • Adaptive routing in LiteLLM can improve reliability and cost efficiency by steering traffic away from degraded or overloaded model deployments automatically.
  • The addition of Moonshot/kimi-k2.6 expands the roster of supported providers, benefiting users who rely on that model's capabilities.
