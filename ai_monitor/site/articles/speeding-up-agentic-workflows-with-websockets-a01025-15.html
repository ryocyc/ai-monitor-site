OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, cutting end-to-end latency in agentic AI workflows by roughly 40% by replacing a series of synchronous HTTP calls with a single persistent connection.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

OpenAI's engineering team has shipped WebSocket support in the Responses API, a change designed to address a fundamental bottleneck that emerged as model inference speeds climbed. In a detailed post published April 22, 2026, engineers Brian Yu and Ashwin Nathan explained that faster inference on newer hardware meant the cumulative overhead of repeated HTTP handshakes and conversation-state reprocessing had become a larger share of total latency. The solution was a persistent WebSocket connection that keeps reusable state cached server-side instead of re-transmitting it with every request.

The engineering work involved caching rendered tokens, prior input and output items, tool definitions, and sampling artifacts in memory for the lifetime of a WebSocket connection. Safety classifiers and request validators were also updated to process only new input rather than re-evaluating the full conversation history on every turn. These changes let Codex and other agentic systems send only new information over the wire, pausing to execute tool calls and resuming inference without rebuilding context from scratch. The result was a reported 45% improvement in time-to-first-token on top of broader API performance gains.

The impact showed up quickly in production. Codex migrated the majority of its Responses API traffic to WebSocket mode and, together with the new GPT-5.3-Codex-Spark model running on Cerebras hardware, pushed inference throughput from roughly 65 tokens per second to over 1,000 tokens per second, with bursts reaching 4,000 TPS. Third-party developers reported similar gains: Vercel's AI SDK saw latency drop by up to 40%, Cline's multi-file workflows ran 39% faster, and Cursor reported up to 30% speed improvements for OpenAI models.

Why this matters

  • As model inference grows faster, the infrastructure layer around it—APIs, networking, state management—has become the new bottleneck, and this work demonstrates a concrete fix for that trade-off.
  • A 40% latency reduction in agentic workflows directly affects developer experience for coding assistants, multi-step automation tools, and any agent that relies on repeated back-and-forth with an LLM API.
  • The familiar `response.create` input shape was preserved, meaning existing integrations can adopt WebSocket mode without rewriting their API interaction code.
