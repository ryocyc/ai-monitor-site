llama.cpp Release b8911 Brings Major Hexagon Profiler Improvements

Release b8911 of llama.cpp lands a sweeping set of changes to its Hexagon (Qualcomm DSP) profiler, including async profiling, configurable PMU counters, and a new post-processing tool.

GitHub llama.cpp Releases 2026-04-24 01:24 UTC Key: github-llamacpp-releases::identity::b8911 Confidence: strong Mode: claude

Article body

Release b8911 of llama.cpp, published on 24 April via GitHub Actions, is entirely devoted to improving the Hexagon backend's profiling subsystem. The headline addition is support for both basic and extended operation profiling on Qualcomm DSP hardware (PR #22269), along with restored PMU (Performance Monitoring Unit) support and a redesigned op-queuing mechanism with separate profile descriptors.

The release renames the old "opmask" concept to "opstage" and eliminates the "opsync" wrapper, citing that the profiler is now fully async and the old naming was misleading. A new configurable profiler mode makes PMU counter collection optional, since reading those counters on older devices carries a meaningful performance cost. The release also introduces interface hooks allowing the host to enable the profiler externally.

A companion tool, `ggml-hexagon-profile.py`, gains the ability to read logs from stdin, simplified output formatting, and environment-variable control over PMU event selection. The `GGML_HEXAGON_PROFILE` environment variable is now documented. Contributions came from Sigbjørn Skjæret, Trivikram Reddy (Qualcomm), and others, touching `ggml/src/ggml-hexagon/htp/htp-ops.h` among other files.

Why this matters

  • Hexagon backend users can now profile with greater fidelity, including async op-stage tracking and configurable PMU events without full overhead on older hardware.
  • The rename from "opmask" to "opstage" and removal of "opsync" reflect a cleaner internal model for how llama.cpp schedules and measures operations on Qualcomm DSPs.
  • A documented environment variable and a post-proc script that accepts stdin make it practical to integrate Hexagon profiling into real-world CI or performance-tuning workflows.
