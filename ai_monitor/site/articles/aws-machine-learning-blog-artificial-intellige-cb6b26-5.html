AWS Security Agent and DevOps Agent Exit Preview, Now Generally Available

AWS has moved its Security Agent and DevOps Agent out of preview and into general availability, marking the full launch of a class of AI capabilities it calls frontier agents.

AWS Machine Learning Blog 2026-04-23 16:32 UTC Key: aws-machine-learning-blog::identity::aws-machine-learning-blog-artificial-intellige Confidence: weak Mode: claude

Article body

AWS has announced the general availability of two autonomous AI agents: AWS Security Agent and AWS DevOps Agent. Both products were introduced as frontier agents at re:Invent and have now completed their preview period, becoming officially available for broader use. The Security Agent handles on-demand penetration testing, while the DevOps Agent assists with cloud operations tasks.

AWS describes frontier agents as autonomous systems capable of working independently toward defined goals, scaling to handle multiple concurrent tasks, and running persistently for hours or days without requiring constant human oversight. The company says these agents are changing how organizations secure and operate software at scale.

During the preview phase, customers and partners used the Security Agent and reported that it helped uncover vulnerabilities. The full scope of capabilities, pricing details, and regional availability were not confirmed at time of publication, as the original announcement page returned a 404 error.

Why this matters

  • General availability signals AWS is confident enough in these agents to recommend enterprise-wide deployment, not just limited trials.
  • The agents represent a shift toward autonomous, long-running AI systems in security and cloud operations — a notable departure from one-shot AI assistance.
  • Preview users already applied Security Agent to real penetration testing work, suggesting these tools are past the concept stage and in active operational use.
