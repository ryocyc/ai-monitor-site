OpenAI Codex CLI Gains Unix Socket Transport, Expanded Plugin Management, and Deeper AWS/Bedrock Integration

OpenAI's flagship AI coding agent shipped version 0.125.0 with a substantive update to its app-server integration layer, adding Unix socket transport, improved plugin lifecycle management, and tighter AWS/Bedrock account-state visibility for application clients.

Article body

OpenAI has published version 0.125.0 of Codex, its command-line coding agent, featuring six new capabilities in the app-server integration layer alongside several bug fixes. The release, dated April 24, 2026, follows a rapid series of alpha pre-releases (v0.126.0-alpha.1 through alpha.3) that have appeared over the subsequent two days, indicating active development on the next milestone.

Among the headline additions, app-server integrations now support Unix socket transport, pagination-friendly resume and fork operations, sticky environments, and remote thread configuration and store plumbing. Plugin management has been expanded to allow installation of remote plugins and upgrades of configured marketplaces. The release also consolidates permission profile handling so that permissions now persist consistently across TUI sessions, user turns, MCP sandbox state, shell escalation, and app-server APIs. On the model-provider side, version 0.125.0 shifts model discovery ownership to providers themselves and exposes AWS and Bedrock account state directly to application clients. A new `--json` flag for `codex exec` surfaces reasoning-token usage, and rollout tracing now records tool, code-mode, session, and multi-agent relationships with a debug reducer command for inspection.

For developers and teams using Codex as a CI/CD or developer-assistance layer, the Unix socket transport and sticky environment support reduce friction when embedding Codex into existing server-side workflows. The expanded plugin marketplace support makes it easier to extend Codex's capabilities without manual configuration. The new tracing and reasoning-token reporting features address observability needs for teams integrating Codex programmatically into larger agentic pipelines.

Why this matters

  • Unix socket transport and sticky environments make Codex easier to embed in server-side developer workflows without workarounds for network or session continuity.
  • Remote plugin installation and marketplace upgrades expand Codex's extensibility for teams that need domain-specific tooling integrated into the agent.
  • Reasoning-token reporting in `codex exec --json` and multi-agent rollout tracing give MLOps and platform teams the visibility needed to monitor and debug Codex in production pipelines.
