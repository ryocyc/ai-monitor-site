Gradio Spaces Now Serve /agents.md Endpoints That AI Coding Agents Can Read Directly

Hugging Face has updated every Gradio Space so that it automatically exposes an /agents.md endpoint, letting AI coding agents such as Claude Code, Codex, or Pi discover and interact with the Space without any manual setup.

Hugging Face Changelog 2026-04-23 11:17 UTC Key: hugging-face-changelog::identity::hugging-face-changelog-hugging-face-changel Confidence: moderate Mode: claude

Article body

When you point a compatible AI coding agent at a Gradio Space, the agent now reads a machine-readable /agents.md endpoint to understand the Space's API surface and call it directly. The endpoint is served automatically — no additional configuration is required from the Space owner beyond what Gradio already provides.

The change is aimed at developers who want their Spaces to be immediately usable by AI agents in automated or agentic workflows. Rather than requiring the agent to infer behavior from a live UI, the /agents.md file provides structured, programmatic context in a standard format that multiple agents can consume.

Documentation for the feature has been published on Hugging Face's site, covering how the endpoint is structured and how different agents can point to it. The changelog entry received around 100 upvotes on the day it went live.

Why this matters

  • Space owners no longer need to build custom agent-integration layers; the endpoint ships with every new Space by default.
  • AI coding agents like Claude Code, Codex, and Pi can now interact with Gradio Spaces through a structured, machine-readable description rather than trying to parse a live UI.
  • This lowers the barrier for building agentic pipelines that compose multiple Hugging Face-hosted tools and models.
