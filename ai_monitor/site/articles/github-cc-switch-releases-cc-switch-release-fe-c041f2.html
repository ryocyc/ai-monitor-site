CC Switch v3.14.1 Ships Tray Usage Counts and Codex OAuth Reverse-Proxy Fixes

CC Switch v3.14.1 adds real-time tray usage display for Claude, Codex, and Gemini providers, alongside targeted Codex OAuth reverse-proxy stability improvements and the removal of the Hermes config health scanner.

GitHub cc-switch Releases 2026-04-23 13:23 UTC Key: github-cc-switch-releases::identity::github-cc-switch-releases-cc-switch-release-fe Confidence: strong Mode: claude

Article body

The v3.14.1 release introduces tray usage visibility for the current provider, surfacing cached subscription summaries and usage-script summaries directly in system tray submenus with color-coded utilization markers. For Chinese coding-plan hosts such as Kimi, Zhipu, and MiniMax, the tray additionally renders a 5-hour and weekly window layout in the same style used by official subscription badges, with the worst utilization level driving the emoji indicator. Creating a Claude provider whose base URL matches a known coding-plan host now auto-injects the usage script metadata, lighting up the tray without manually opening the Usage Script modal.

Several Codex OAuth reverse-proxy stability issues are addressed in this release. Client-provided session IDs are now used as both the prompt cache key and the Codex session header, eliminating UUID-driven cache churn. Non-streaming Anthropic clients now receive proper JSON responses even when the upstream ChatGPT Codex forces OpenAI Responses SSE. Stream Check probes are now built with the same `store: false`, encrypted reasoning, and FAST mode settings as production requests, closing the gap that caused the tool to report failures for otherwise functional sessions. A new explicit FAST mode toggle is also introduced, allowing users to opt into `service_tier="priority"` on Codex OAuth-backed providers at their own discretion.

The Hermes config health scanner — including its warning banner, `scan_hermes_config_health` command, `HermesHealthWarning` type, and associated payload — has been removed entirely. This refocusses the Hermes surface on provider display, switching defaults, memory editing, and launching the Web UI, delegating deep configuration health judgment back to Hermes itself. The release covers 13 commits across 48 files, with contributions from @TuYv, @JesusDR01, @majiayu000, and @xpfo-go.

Why this matters

  • System tray now surfaces live provider usage data, reducing the need to open in-app modals to check quotas and utilization markers.
  • Codex OAuth reverse-proxy stability fixes resolve session routing churn, SSE aggregation mismatches, and Stream Check parity issues that could cause false negatives in working environments.
  • Removing the Hermes config health scanner simplifies the UI and clarifies that deep configuration health is Hermes's own responsibility rather than a shared concern.
