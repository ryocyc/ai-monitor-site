Hugging Face Gradio Spaces Now Serve a Machine-Readable /agents.md Endpoint for AI Coding Agents

Hugging Face has quietly added an auto-served /agents.md endpoint to every Gradio Space, giving AI coding agents like Claude Code, Codex, and Pi a plug-and-play way to discover and call Space APIs without manual configuration.

Article body

As of April 17, 2026, every Gradio Space hosted on Hugging Face now automatically exposes an /agents.md endpoint containing a machine-readable API description. Coding agents that point to a Space — such as Claude Code, Anthropic's Codex, or the Pi agent — can read this description and invoke the Space's functionality without any additional setup or custom tooling on the host's side.

The endpoint is served natively by Gradio, meaning Space creators receive this capability without modifying their applications. According to the changelog, the format is designed to be consumed directly by agents, allowing them to infer how to interact with a Space's inputs and outputs automatically. The feature builds on Gradio's existing role as the dominant platform for hosting interactive ML demo applications on Hugging Face.

For builders of AI coding agents and agentic workflows, the change reduces the integration friction of connecting to third-party or community-built Gradio Spaces. Rather than reverse-engineering API interfaces or relying on hardcoded wrappers, agents can now discover Space capabilities on demand. This positions Gradio Spaces as a more composable part of the agent tooling ecosystem, with implications for how developers scaffold multi-agent pipelines that pull in community-built models and demos.

Why this matters

  • Gradio Spaces on Hugging Face host thousands of community and enterprise ML demos — giving coding agents access to this ecosystem dramatically lowers the barrier to using third-party tools and models in agentic workflows.
  • Space creators get agent-compatible API discovery for free, with no changes required to their existing applications, which could accelerate adoption of the feature across the platform.
  • The changelog entry reflects a broader trend toward agent-native infrastructure: services that expose machine-readable interface descriptions so AI agents can integrate them without bespoke glue code.
