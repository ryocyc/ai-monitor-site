LangGraph Prebuilt 1.0.11 Brings Flexible Tool Returns and ToolRuntime Exposure

LangChain AI released langgraph-prebuilt version 1.0.11 on April 24, adding support for ToolNode tools that return structured lists of commands or messages.

GitHub LangGraph Releases 2026-04-24 18:19 UTC Key: github-langgraph-releases::identity::langgraph-prebuilt1011 Confidence: strong Mode: claude

Article body

LangChain AI has published langgraph-prebuilt version 1.0.11 to GitHub, marking a minor release that advances the prebuilt agent toolkit's tool-calling capabilities. The update, timestamped 2026-04-24T18:18:45Z, introduces a new feature allowing ToolNode tools to return a list of Command or ToolMessage objects — a flexibility that gives developers finer control over agent orchestration logic. The release also includes a new capability to expose available tools on the ToolRuntime object, making tool discovery more transparent at runtime. Under the hood, LangGraph itself was bumped from version 1.1.8 to 1.1.9 as part of this release cycle.

Why this matters

  • Developers building multi-step agents can now structure ToolNode responses with greater granularity, enabling more complex dialogue and control-flow patterns without custom wrappers.
  • Exposing available tools on ToolRuntime improves introspectability, helping teams debug and document which tools an agent has access to at any given point.
  • The release is a straightforward minor version update with minimal migration risk, but teams currently on prebuilt==1.0.10 should evaluate the new return-type contract for ToolNode.
