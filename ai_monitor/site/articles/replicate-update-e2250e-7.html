Replicate Publishes Agent Skills for Claude Code, OpenAI Codex, and Compatible Tools

Replicate now offers Agent Skills — markdown instruction files that give coding assistants structured guidance on discovering, comparing, and calling its models via the API, as well as tailored prompting techniques for image and video ge...

Replicate Changelog 2026-04-22 23:49 UTC Key: replicate-changelog::identity::replicate-update Confidence: strong Mode: claude

Article body

The new Agent Skills are markdown files installed directly into a coding assistant's workflow using a single command: `npx skills add replicate/skills`. Once added, they provide the assistant with expert-level knowledge about which Replicate models to choose for a given task, how to write effective prompts for image and video generation, and what tradeoffs to weigh — knowledge that would otherwise have to be supplied manually in a prompt.

The skills follow the open Agent Skills specification and are compatible with Claude Code, OpenAI Codex, OpenCode, and other tools that support the spec. Replicate describes the skills as complementary to its existing MCP server: while MCP grants an assistant API-level tooling, the skills layer on top to explain *how* to use those tools well, covering model discovery and comparison in addition to execution.

Replicate provides a reference guide and a GitHub repository for the skills. The April 21 announcement is part of a broader pattern on the platform — recent changelog entries also cover fallback model routing for Nano Banana Pro, MCP server auto-discovery via the official MCP Registry, and improved documentation tooling for language models.

Why this matters

  • Reduces the trial-and-error needed to get good outputs from Replicate's model catalog, since the assistant already knows prompting patterns and tradeoffs.
  • Standardizes how coding assistants interact with Replicate through an open spec, making the integration portable across tools rather than locked to one assistant.
  • Makes Replicate's MCP server more immediately useful by pairing low-level API access (via MCP) with high-level usage guidance (via Skills), lowering the barrier for automated AI workflows.
