xAI Launches Grok Speech-to-Text and Text-to-Speech APIs with Multilingual Support

xAI has released two standalone audio APIs—Grok Speech to Text and Grok Text to Speech—bringing its speech technology to developers outside the Grok chatbot for the first time.

xAI Blog 2026-04-22 23:49 UTC Key: xai-blog::identity::grok-stt-and-tts-apis Confidence: strong Mode: claude

Article body

The APIs, announced April 17, 2026, are built on the same speech stack used by Grok Voice, Tesla vehicles, and Starlink customer support. Grok STT offers both a REST API for batch transcription and a WebSocket API for real-time streaming, and the company benchmarks it against ElevenLabs, Deepgram, and AssemblyAI. On a Word Error Rate benchmark across phone calls, video and podcasts, meetings, and telephony, Grok STT scored 6.9% overall—lower, and thus better, than all three competitors, with particularly strong results on phone call entity extraction at 5.0%.

The Text to Speech API distinguishes itself through inline speech tags such as `[laugh]`, `[sigh]`, `[whisper]`, and emotion and pace modifiers, which xAI says allow developers to add natural prosody without complex markup. Both APIs support 25 or more languages. Grok STT also includes speaker diarization with word-level speaker IDs and intelligent Inverse Text Normalization that converts raw spoken output into structured text with proper numbers, dates, and currencies.

Pricing is tiered by use case: STT batch transcription costs $0.10 per audio hour and streaming transcription costs $0.20 per hour, while TTS is priced at $4.20 per million characters. All rates are available in the xAI API console alongside current rate limits. Links to the API documentation and the voice playground are included in the announcement.

Why this matters

  • xAI is moving its speech technology out of the Grok chatbot and into standalone developer endpoints for the first time, competing directly with established speech API providers.
  • Grok STT's benchmark figures show a meaningful accuracy gap over leading rivals on phone call and entity-heavy tasks, which are common in enterprise workflows like compliance, CRM, and support analytics.
  • The inclusion of speech tags for TTS gives developers a lightweight way to add expressive delivery—something most TTS APIs lack without custom model fine-tuning.

Source note

  • The full announcement was published on xAI's blog on April 17, 2026. It includes an interactive voice demo, comparison benchmarks, and links to the xAI API console and developer documentation for both endpoints.
