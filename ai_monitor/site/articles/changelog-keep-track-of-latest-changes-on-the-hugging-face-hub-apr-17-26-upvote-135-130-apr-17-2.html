Hugging Face Auto-Generates /agents.md Endpoints for All Gradio Spaces

Every Gradio Space on Hugging Face now automatically serves a machine-readable /agents.md endpoint, allowing AI coding agents like Claude Code, Codex, and Pi to discover and use the space without manual configuration.

Article body

Hugging Face has introduced a zero-configuration way for AI coding agents to interact with Gradio Spaces. As of April 17, 2026, every Gradio Space automatically exposes an /agents.md endpoint — a machine-readable API description that agents can read and act on directly, without any prior setup or custom integration. Developers simply point their agent at the space and it figures out how to use the underlying functionality on its own. The endpoint is designed to be read by any AI agent capable of parsing structured documentation. According to Hugging Face's own changelog, agents including Claude Code, Codex, and Pi are already compatible with this format. This effectively turns any Gradio Space into a plug-and-play tool for AI agents, removing a layer of friction that previously required bespoke wiring for each agent-space pair. Hugging Face has published documentation for developers who want to understand or extend how the /agents.md format works. The feature was posted on April 17 and has already received significant community engagement.

Why this matters

  • AI coding agents can now use any Gradio Space without per-agent or per-space custom code, dramatically lowering the barrier to multi-agent workflows.
  • Developers publishing Gradio Spaces gain instant agent compatibility with no additional work beyond the standard space creation process.
  • The standardisation of a machine-readable /agents.md format could drive broader ecosystem convergence around AI tool-calling conventions.
