Agent Skills bring expert model knowledge to Claude Code, Codex, and OpenCode

Replicate now ships "agent skills" — markdown instruction files that let coding assistants like Claude Code and OpenAI Codex work more effectively with its image, video, and general-purpose AI models.

Replicate Changelog 2026-04-22 23:49 UTC Key: replicate-changelog::identity::replicate-update Confidence: strong Mode: claude

Article body

Replicate has introduced agent skills, a set of markdown instruction files that give AI coding assistants structured knowledge about working with its platform. The skills cover model discovery, model comparison, API execution, and prompting techniques for image and video generation. They are installed with a single npm command — npx skills add replicate/skills — which drops the files into a project and configures the coding assistant automatically.

The skills follow the open Agent Skills spec and are compatible with Claude Code, OpenCode, OpenAI Codex, and other tools that honor the spec. Replicate frames them as a complement to its existing MCP server: where MCP hands a coding assistant the raw API tools to make calls, the skills provide the judgment — which model to pick, how to phrase a prompt, and what tradeoffs to weigh. The two systems are designed to work together rather than replace each other.

Agent skills are the only changelog entry dated April 21, 2026. Two other entries — a fallback model option for the Nano Banana Pro image model (falling back to ByteDance Seedream 5.0 lite on API capacity) and MCP server auto-discovery via the official MCP Registry — appear to be older additions, reflecting a broader push to make Replicate more accessible to automated and agentic workflows.

Why this matters

  • Coding assistants can now reason about Replicate's model catalog rather than treating it as a black box — choosing the right model and writing effective prompts without manual lookup.
  • The open Agent Skills spec means tools built for it work across platforms, not just Replicate, reducing lock-in for teams using multiple AI services.
  • Pairing skills with the MCP server creates a layered approach where automation tools get both the capability to act (MCP tools) and the knowledge to act well (skills).
