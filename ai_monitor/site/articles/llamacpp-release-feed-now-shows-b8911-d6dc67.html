llama.cpp Release b8911 Revamps Hexagon DSP Profiling

The llama.cpp project tagged release b8911 on April 24, 2026, delivering a sweeping set of improvements to its Qualcomm Hexagon DSP backend with a new async profiler, configurable PMU counters, and a post-processing tool.

GitHub llama.cpp Releases 2026-04-24 01:24 UTC Key: github-llamacpp-releases::identity::b8911 Confidence: strong Mode: claude

Article body

Release b8911 of llama.cpp, published on April 24, 2026 via GitHub Actions and signed with the project's verified GPG key, centers almost entirely on the Hexagon DSP backend. The headline change is a rebuilt profiling pipeline: the existing op profiling is restored and extended, PMU (Performance Monitoring Unit) counters are enabled, and a new async profiler replaces the older blocking approach. opsync has been removed and the confusingly named "opmask" has been renamed "opstage" to better reflect its role in the pipeline.

Several changes make the profiler more developer-friendly in practice. Profiler mode is now optional on older hardware, since reading PMU counters on those devices carries a real performance cost. The release adds interface hooks so the host can enable the profiler externally, and the GGML_HEXAGON_PROFILE environment variable is now documented for configuration. Output formatting has been cleaned up and a companion post-processing script (hex-prof) is included, capable of reading logs from stdin.

Beyond profiling, the release also restores HTP_OPMASK_QUEUE and ensures that OPMASK_SKIP_COMPUTE is properly honored in hmx-matmul operations. The opbatch queue handling has been refactored, and several linter warnings in the htp-ops header have been fixed. Co-authorship from Qualcomm contributors Sigbjørn Skjæret and Trivikram Reddy signals direct vendor involvement in these changes.

Why this matters

  • The async profiler and configurable PMU mode give developers a way to profile Hexagon performance without distorting the results they are trying to measure.
  • Renaming "opmask" to "opstage" and adding documentation for GGML_HEXAGON_PROFILE make the backend easier to understand and tune.
  • Vendor-adjacent contributors (Qualcomm) appearing in the commit history suggests these Hexagon improvements are becoming a more intentional, supported part of the project.
