llama.cpp Flips HIP Graphs to Default-On for AMD GPUs

The llama.cpp project has re-enabled HIP graph execution by default for AMD hardware, citing measurable improvements in ROCm performance.

GitHub llama.cpp Releases 2026-04-23 02:36 UTC Key: github-llamacpp-releases::identity::github-llamacpp-releases-llamacpp-release-fee Confidence: strong Mode: claude

Article body

Release b8893 of llama.cpp changes the default for GGML_HIP_GRAPHS to "on" for AMD GPUs. The project notes that an earlier decision to disable HIP graphs by default (in PR #11362) was made because the performance impact was negative at the time. Since then, both ROCm and llama.cpp's own graph usage and construction have improved enough to flip that default. The release ships a broad set of pre-built binaries across platforms and backends, including CUDA 13.1 and Vulkan builds for Windows, ROCm 7.2 binaries for Ubuntu, and several openEuler variants targeting Intel and ARM silicon. The release page also lists macOS Apple Silicon builds with optional KleidiAI support, iOS XCFramework binaries, and Android arm64 CPU builds.

Why this matters

  • AMD GPU users running llama.cpp on ROCm should see performance gains from graph execution without needing to manually enable it.
  • The release consolidates support for a wide hardware matrix — spanning CUDA, Vulkan, HIP, SYCL, and OpenVINO — making multi-backend deployment more consistent.
  • The KleidiAI-enabled macOS Apple Silicon build signals ongoing optimization work for ARM-based inference beyond the traditional CUDA path.
