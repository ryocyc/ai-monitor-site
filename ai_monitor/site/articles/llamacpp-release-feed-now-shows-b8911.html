llama.cpp Release b8911: Hexagon backend profiler gets major rework

Release b8911 of llama.cpp delivers a substantial overhaul of Hexagon processor profiling, including async operation tracking, configurable PMU counters, and a new post-processing tool.

GitHub llama.cpp Releases 2026-04-24 01:24 UTC Key: github-llamacpp-releases::identity::b8911 Confidence: strong Mode: claude

Article body

Release b8911 of llama.cpp, published 24 April, focuses heavily on the Hexagon backend's profiling infrastructure. The update adds support for both basic and extended operation profiling via a new async profiler that replaces the older opsync mechanism, which has been removed as no longer necessary. Operation masking has also been renamed to "opstage" to better reflect its updated role.

PMU (Performance Monitoring Unit) support has been restored and made optional on older devices where reading PMU counters carries a performance cost. Users can now configure profiler mode and set PMU events via the GGML_HEXAGON_PROFILE environment variable. Profiler output has been simplified, and a new command-line post-processing tool (hex-prof) is included to handle log analysis, including reading from stdin.

The release ships prebuilt binaries across a wide range of targets: macOS (arm64, Intel), Linux (various CPUs, Vulkan, ROCm, OpenVINO, SYCL), Windows (CPU, CUDA, Vulkan, SYCL, HIP), Android arm64, and openEuler on x86 and aarch64 with ACL Graph support.

Why this matters

  • The async profiler and configurable PMU mode give developers more granular control over performance tuning on Qualcomm Hexagon hardware.
  • Renaming opsync to opstage and adding separate profile descriptors reflect a clearer internal architecture that may improve maintainability.
  • Prebuilt binaries for openEuler with ACL Graph targets signal continued expansion of hardware platform support.
