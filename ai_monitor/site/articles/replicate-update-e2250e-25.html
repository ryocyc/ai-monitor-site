Replicate Now Publishes Agent Skills for AI Coding Assistants

Replicate has released a set of markdown-based instruction files designed to give AI coding assistants expert-level knowledge about working with its models and API.

Replicate Changelog 2026-04-22 23:49 UTC Key: replicate-changelog::identity::replicate-update Confidence: strong Mode: claude

Article body

Replicate now publishes Agent Skills, a collection of markdown instruction files that give coding assistants expert knowledge about working with AI models on the platform. The skills cover model discovery, comparison, and execution via the API, along with detailed prompting techniques for image generation and video generation models. They follow the open Agent Skills spec and are designed to work with Claude Code, OpenCode, OpenAI Codex, and other compatible tools. Installing them is a single command: npx skills add replicate/skills, which pulls all skills into a project and configures them for the target assistant automatically.

The skills are positioned as a complement to Replicate's existing MCP server. Where MCP provides API tools to a coding assistant, the new skills layer on top to teach assistants how to use those tools well — which models to pick, how to structure prompts, and what tradeoffs to consider. The changelog notes that more details can be found in the agent skills reference or the GitHub repository.

The release follows a run of smaller updates across the platform, including a fallback model option for Nano Banana Pro (now falling back to ByteDance Seedream 5.0 lite when Google's API hits capacity), auto-discovery support for the MCP server via the official MCP Registry, and a new source query parameter on the list predictions endpoint to isolate predictions made through the web interface.

Why this matters

  • Agent Skills bring structured, model-specific guidance directly into AI coding assistants without requiring custom prompting or external documentation.
  • The open Agent Skills spec means tooling built around it can be reused across different platforms, not just Replicate.
  • The distinction between MCP (tools) and Skills (knowledge) clarifies how Replicate is building a layered integration story for developers and AI agents alike.
