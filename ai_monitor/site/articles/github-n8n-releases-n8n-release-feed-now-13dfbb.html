n8n Stable Reaches v2.17.6 with User Search Bug Fix

The n8n workflow automation platform has tagged its stable release as v2.17.6, addressing a bug in the user search feature within ProjectSettings.

Article body

The n8n project's stable release tag was updated on April 23, 2026 to version 2.17.6. The single bug fix in this release targets user search functionality inside ProjectSettings, with the change tied to pull request #28949 (commit d388990). The release was published by the n8n-assistant account and signed with GitHub's verified signature.

Simultaneously, n8n v2.18.2 was published on the same day, adding a MiniMax standalone vendor node and additional bug fixes. v2.17.6 and v2.18.2 share the same user search correction across different release tracks, with v2.18.2 carrying the fix in a separate PR (#28948, commit c1ab423).

This is a modest patch by n8n standards. The stable channel addresses a narrowly scoped UI/workflow issue rather than a systemic problem, while the faster-moving feature release (v2.18.2) carries the more substantial new node addition.

Why this matters

  • Teams relying on ProjectSettings user search can expect corrected behavior after updating to v2.17.6 or later.
  • The same user search fix appears in v2.18.2, meaning users on the latest feature release also receive the correction.
  • A new standalone MiniMax vendor node debuted in v2.18.2 on the same day, expanding n8n's integration ecosystem.
