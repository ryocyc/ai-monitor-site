OpenAI Adds WebSocket Support to Responses API, Cutting Agentic Workflow Latency by 40%

OpenAI has introduced WebSocket mode in its Responses API, allowing developers to maintain a persistent connection that reuses conversation state and reduces redundant processing across multi-turn agent loops.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When OpenAI launched GPT-5.3-Codex-Spark—a coding model built for speed—the team quickly realized that the Responses API itself had become the bottleneck. The model could run inference at over 1,000 tokens per second on specialized Cerebras hardware, but each Codex task still required dozens of back-and-forth requests that accumulated into minutes of waiting. To bridge that gap, OpenAI turned to a persistent WebSocket transport that keeps reusable context alive across a single long-running connection.

The core idea is straightforward: instead of re-sending the full conversation history with every follow-up request, the server caches prior response state in memory for the lifetime of the WebSocket connection. Safety classifiers, tokenizers, and model-routing logic all operate only on new input rather than reprocessing the entire history each time. Non-blocking tasks like billing are overlapped with subsequent requests, allowing multiple stages of a request pipeline to run concurrently instead of sequentially.

The results are already showing up in developer tools. Codex migrated the majority of its Responses API traffic onto WebSocket mode within weeks of launch. Vercel's AI SDK saw latency drop by up to 40%, Cline's multi-file workflows ran 39% faster, and Cursor reported up to 30% speed gains for OpenAI models running there. OpenAI calls WebSocket mode one of the most significant new capabilities in the Responses API since its initial launch in March 2025.

Why this matters

  • Agentic AI workflows—where a model repeatedly calls tools, waits for results, and continues—are only as fast as the underlying transport layer allows; until now, that layer was adding hidden overhead that grew worse with longer conversations.
  • Sustained improvements in model inference speed only translate to user-facing gains if the surrounding API infrastructure can keep pace—WebSocket mode is designed explicitly to close that gap.
  • Adoption across Codex, Vercel, Cline, and Cursor means developers building on those platforms automatically get faster agentic experiences without rewriting their integrations.
