llama.cpp Release b8907 Updates LibreSSL Dependency to 4.3.1

The llama.cpp project tagged release b8907, bumping its LibreSSL vendor dependency to version 4.3.1.

Article body

The ggml-org/llama.cpp project released tag b8907 on 23 April 2026. The sole change in this release is a vendor update bumping LibreSSL to version 4.3.1, tracked as pull request #22285. The commit was signed off by Adrien Gallouët of Hugging Face and verified with GitHub's vigilant mode signature.

The release ships prebuilt binaries across a wide range of platforms: macOS (Apple Silicon with and without KleidiAI, and Intel x64), iOS as an XCFramework, and Linux builds for Ubuntu on x64 and arm64 CPUs with optional Vulkan, ROCm 7.2, OpenVINO, and SYCL acceleration. Windows builds cover CPU, CUDA 12 and 13, Vulkan, SYCL, and HIP backends. Android arm64, openEuler variants, and Ubuntu s390x are also included.

No functional changes to llama.cpp's inference code are associated with this tag. The update brings the bundled LibreSSL version forward, which typically carries security fixes and compatibility improvements for TLS and cryptographic operations used within the project.

Why this matters

  • LibreSSL updates commonly include security patches for TLS and cryptography that affect how llama.cpp handles network requests and secure model downloads.
  • This release propagates the dependency bump across all officially distributed binary archives, so users on macOS, Windows, Linux, iOS, and Android get the same baseline LibreSSL version.
  • The update covers CUDA 13 (with 13.1 DLLs) and ROCm 7.2, indicating continued alignment with the latest GPU compute stacks supported by llama.cpp.
