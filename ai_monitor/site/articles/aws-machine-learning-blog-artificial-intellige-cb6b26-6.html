AWS Security Agent and DevOps Agent Exit Preview, Now Generally Available

Article body

AWS Security Agent and AWS DevOps Agent are now generally available, the company announced on March 31, 2026. The two products represent a class of AI capabilities AWS first introduced at re:Invent under the "frontier agents" umbrella. Security Agent handles on-demand penetration testing, while DevOps Agent targets cloud operations tasks. Both are designed to work independently, scaling across concurrent jobs and running for extended periods — hours or days — without requiring a human in the loop.

During the preview period, customers and partners reported that Security Agent provided comprehensive coverage for on-demand penetration testing scenarios. The broader promise of frontier agents, as AWS frames it, is that they can tackle goals autonomously, scale massively across parallel tasks, and persist through long-running workloads without constant supervision.

The launch signals AWS's push to embed autonomous AI deeper into security and DevOps workflows, positioning these agents as a new layer of infrastructure rather than a one-off tool. The company says these systems are already changing how teams secure and operate software at scale.

Why this matters

  • Penetration testing traditionally requires significant human coordination; Security Agent automates that workflow end-to-end.
  • DevOps Agent's long-duration, persistent operation allows it to manage cloud infrastructure tasks that previously demanded continuous human attention.
  • The GA launch means any AWS customer can now access these frontier agents without joining a preview program.
