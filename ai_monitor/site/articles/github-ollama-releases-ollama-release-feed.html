Ollama v0.21.1 Released with Kimi CLI and MLX Performance Improvements

Ollama v0.21.1 stabilises a new Kimi CLI launcher and several MLX runner optimisations, marking the latest stable release from the ollama project.

GitHub Ollama Releases 2026-04-22 12:43 UTC Key: github-ollama-releases::identity::github-ollama-releases-ollama-release-feed- Confidence: moderate Mode: claude

Article body

Ollama v0.21.1 has been promoted to stable as of 22 April 2026. The release includes a new Kimi CLI integration, accessible via `ollama launch kimi --model kimi-k2.6:cloud`, which enables multi-agent execution for long-horizon tasks using the Kimi K2.6 cloud model. The update also introduces logprobs support for compatible models in the MLX runner, alongside a fused top-P and top-K sampling pass and repeat penalties applied within the sampler for faster MLX performance.

Tokenization for the MLX runner has been moved into request handler goroutines, improving prompt tokenization throughput, while array management has gained better thread safety. A fused sigmoid router head has been introduced for GLM4 MoE Lite, delivering a performance uplift for that architecture. Two user-facing bugs were also resolved: the macOS app model picker no longer shows stale models after chat switching, and structured outputs now work correctly for Gemma 4 when `think=false`.

Why this matters

  • MLX users on Apple Silicon see concrete sampling speedups from fused top-P/top-K and improved tokenization threading.
  • The new Kimi CLI launcher makes a multi-agent cloud model directly accessible through the Ollama interface for the first time.
  • Two macOS app bugs fixed — stale model picker and Gemma 4 structured output — directly improve day-to-day usability for Apple-platform users.
