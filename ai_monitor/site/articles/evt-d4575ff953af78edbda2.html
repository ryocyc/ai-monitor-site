n8n 2.19.0 Ships AI-Builder Stability Fixes and Credential Updates

n8n's latest release tightens up its AI-builder tooling with fixes for duplicate workflow creation, workflow identity persistence, and credential authentication across five integrations.

Article body

n8n, the open-source workflow automation platform, released version 2.19.0 on April 28, 2026, landing a concentrated set of bug fixes across its AI-builder components and credential management layer. The release addresses six distinct AI-builder issues alongside authentication fixes for Twilio, Pipedrive, Asana, Freshdesk, and Workable integrations.

The AI-builder changes target stability in multi-user and concurrent scenarios: the team patched duplicate workflow creation triggered by parallel submits in instance AI, stopped empty default values from being incorrectly treated as something else, and preserved workflow identity when a workflow is rebuilt or patched. Routing nodes also receive additional key handling, and schema improvements land for button, icon, callout, and workflowSelector components in the builder UI.

For builders and teams deploying n8n in production, the 2.19.0 release removes several sharp edges that could cause workflow instability or data duplication in AI-augmented automation pipelines. The credential auth fixes across five services mean fewer broken connections during routine re-authentication flows, while the AI-builder improvements make the platform safer to use in concurrent or multi-user environments where workflows are dynamically generated or modified at runtime.

Why this matters

  • Duplicate workflow creation on parallel submits is a data-integrity risk in any production automation environment; the fix closes it directly.
  • Workflow identity persistence across rebuild and patch operations means developers can safely update AI-generated workflows without losing references or causing downstream failures.
  • Credential auth fixes for Twilio, Pipedrive, Asana, Freshdesk, and Workable restore reliability for some of n8n's most commonly used enterprise integrations.
