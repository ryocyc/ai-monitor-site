OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, enabling persistent connections that cut end-to-end latency for agentic workflows by up to 40 percent.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When you ask Codex to fix a bug, it scans your codebase, reads files, makes edits, and runs tests—cycling through dozens of sequential API calls each time. As inference speeds have climbed, the cumulative overhead of those repeated HTTP handshakes became the new bottleneck. OpenAI's Responses API now handles these workflows over a persistent WebSocket connection, caching conversation state, token artifacts, and tool definitions in memory so that each follow-up request only sends new information rather than rebuilding context from scratch.

The engineering team rethought the transport layer after discovering that earlier optimizations—caching rendered tokens, reducing network hops, and streamlining safety classifiers—improved time-to-first-token by roughly 45 percent but still left API overhead disproportionately large relative to the speed of the new model. The solution was to model an entire agentic rollout as a single long-running Response over WebSockets, blocking the sampling loop after a tool call and resuming only when the client sends back the tool result.

For the shipped version, OpenAI settled on keeping the familiar `response.create` input shape and using `previous_response_id` to pull cached state from a connection-scoped in-memory store. This gave developers a drop-in upgrade without rewriting their API integration. The gains were immediate: GPT-5.3-Codex-Spark reached the 1,000 tokens-per-second target with bursts up to 4,000 TPS, while Codex migrated the majority of its Responses API traffic to WebSocket mode within days of launch.

Why this matters

  • Agentic AI tools like Codex, Cursor, and Cline now run significantly faster without any changes to their prompts or logic.
  • The persistent-connection approach eliminates redundant tokenization and context-rebuilding that had accumulated as inference got faster.
  • OpenAI's open adoption by Vercel's AI SDK suggests the pattern could become a standard capability across the broader AI tooling ecosystem.
