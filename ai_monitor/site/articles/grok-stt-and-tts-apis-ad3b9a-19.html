Grok Speech to Text and Text to Speech APIs

xAI has released two standalone audio APIs — Grok STT and Grok TTS — built on the infrastructure powering Grok Voice, Tesla vehicles, and Starlink customer support.

xAI Blog 2026-04-22 23:49 UTC Key: xai-blog::identity::grok-stt-and-tts-apis Confidence: strong Mode: claude

Article body

The Grok Speech to Text API offers high-accuracy transcription with low latency through both REST and WebSocket endpoints. It includes word-level timestamps, speaker diarization, multichannel support, and Inverse Text Normalization that converts spoken numbers, dates, and currencies into structured output. A performance comparison table on the page shows Grok STT achieving an overall Word Error Rate of 6.9%, outperforming ElevenLabs (9.0%), Deepgram (11.0%), and AssemblyAI (12.9%) across phone call, video/podcast, meeting, and telephone domains.

The Grok Text to Speech API delivers natural, expressive voices via REST and WebSocket APIs, with fine-grained control through inline speech tags like `[laugh]`, `[sigh]`, `[whisper]`, and emotional modifiers. Both APIs support 25 or more languages with seamless switching during transcription.

Pricing is straightforward and predictable: STT is priced at $0.10 per hour for batch processing and $0.20 per hour for real-time streaming, while TTS costs $4.20 per 1 million characters. Developers can get started via the xAI API console and documentation links provided on the announcement page.

Why this matters

  • xAI is entering a crowded speech API market with benchmark numbers that position Grok STT as the top performer across multiple audio domains.
  • The combination of STT and TTS as standalone endpoints gives developers a single vendor option for both speech conversion needs, with inline speech tags on TTS offering creative control rarely seen at the API level.
  • The flat-rate, usage-based pricing model with no hidden fees contrasts with the tiered or per-request structures of established competitors.
