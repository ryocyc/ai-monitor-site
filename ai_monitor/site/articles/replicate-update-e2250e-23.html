Replicate Publishes Agent Skills for AI Coding Assistants

Replicate now offers structured markdown instruction files designed to give coding assistants — including Claude Code, OpenAI Codex, and OpenCode — expert-level knowledge for working with its model API.

Replicate Changelog 2026-04-22 23:49 UTC Key: replicate-changelog::identity::replicate-update Confidence: strong Mode: claude

Article body

Replicate has published a collection of agent skills: markdown files that encode domain knowledge about its platform for AI coding assistants. The skills cover model discovery, side-by-side comparison, API execution, and prompting strategies specifically for image and video generation models. They follow the open Agent Skills specification, making them compatible with any tool that adheres to that standard.

The skills are installed via a single command — `npx skills add replicate/skills` — which pulls all available skills into a project and configures them for the active coding assistant automatically. The collection is designed to complement, not replace, Replicate's Model Context Protocol (MCP) server. Where MCP provides API-level tools for a coding assistant, agent skills supply the judgment: which model fits a given task, how to phrase prompts effectively, and what tradeoffs to expect.

The release follows an open specification rather than a proprietary format, signalling that Replicate intends interoperability beyond its own ecosystem. Detailed reference documentation and a GitHub repository are available for developers who want to inspect or extend the skill set.

Why this matters

  • The open Agent Skills spec means tooling built for Replicate can potentially work across different AI assistant platforms without rework.
  • The distinction between MCP (tools) and agent skills (knowledge) gives developers a clearer mental model for integrating Replicate into agentic workflows.
  • Installing via `npx skills add` lowers the barrier to entry for developers who want context-aware assistance without manually studying the API documentation.
