Speeding Up Agentic Workflows with WebSockets in the Responses API

OpenAI has added WebSocket support to the Responses API, cutting end-to-end latency for agentic workflows by up to 40% by keeping a persistent connection and caching reusable state instead of rebuilding context with every call.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

Agent loops like the one powering Codex make dozens of back-and-forth requests per task — each one re-sending full conversation history, re-validating inputs, and re-rendering tokens. As inference speed climbed past 1,000 tokens per second on the GPT-5.3-Codex-Spark model (compared to roughly 65 TPS on earlier flagship models), the cumulative overhead of those repeated API calls became the new bottleneck, dwarfing the time saved on inference alone.

The fix was to stop treating each agent step as an independent HTTP request. With WebSocket mode, the server maintains a connection-scoped, in-memory cache of previous response state — prior items, tool definitions, and rendered tokens — so follow-up calls using a `previous_response_id` skip redundant work. Safety classifiers, request validators, and tokenization all process only new input rather than replaying the full conversation history. Critically, the API shape stays the same: developers keep using `response.create` with the same body, just over a persistent WebSocket connection instead of a series of one-off HTTP calls.

Early adoption from Codex, Vercel, Cline, and Cursor has produced measurable gains across the ecosystem. Codex has already moved the majority of its Responses API traffic onto WebSocket mode. Vercel's AI SDK reports up to 40% lower latency, Cline's multi-file workflows run 39% faster, and Cursor's OpenAI-powered features are up to 30% faster. The team describes WebSocket mode as one of the most significant new capabilities in the Responses API since its launch in March 2025.

Why this matters

  • As model inference gets faster, API overhead becomes the limiting factor for real-world agent speed — and WebSocket mode directly addresses that shift.
  • The feature requires no new API shapes or client rewrites, making it accessible to existing Responses API integrations without redesign effort.
  • Third-party tooling across the developer ecosystem — Codex, Vercel, Cline, Cursor — is already reporting significant latency reductions in production.
