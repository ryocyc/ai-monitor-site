OpenAI Adds WebSocket Support to Responses API, Cutting Agentic Workflow Latency by 40%

OpenAI has built WebSocket mode into its Responses API, allowing agent loops to maintain a persistent connection and skip redundant processing, resulting in up to 40% faster end-to-end performance for tasks like code completion.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

OpenAI's engineering team has introduced WebSocket support to the Responses API, a change targeted squarely at the latency bottleneck that emerges when AI agents make dozens of back-and-forth requests in sequence. As inference speeds climb, the cumulative overhead of repeatedly setting up connections and reprocessing full conversation histories becomes the dominant delay—and the new persistent-connection design addresses exactly that.

The core idea is simple: instead of opening a new HTTP connection for each follow-up request, a WebSocket keeps the connection alive and caches conversation state on the server side. When a client sends a `response.create` call with a `previous_response_id`, the server pulls the cached state—prior input and output items, tool definitions, and pre-rendered tokens—rather than rebuilding it from scratch. Safety classifiers and request validators can then process only the new information, not the full history. The same `response.create` body is used throughout, so developers can adopt WebSocket mode without rewriting their integration.

Early feedback from the alpha was strong enough to ship broadly. Codex migrated the majority of its Responses API traffic onto WebSocket mode within days of launch, and the team hit its target of 1,000 tokens per second for GPT‑5.3‑Codex‑Spark, with bursts reaching 4,000 TPS. Integrations across Vercel's AI SDK, Cline's multi-file workflows, and Cursor's editor all reported latency drops in the 30–40% range. OpenAI calls WebSocket mode one of the most significant new capabilities in the Responses API since its launch in March 2025.

Why this matters

  • As inference speeds improve, API overhead becomes the new bottleneck—and persistent connections with in-memory state caching offer a clean solution.
  • Developers can enable WebSocket mode without changing their API input and output shapes, making adoption low-friction.
  • Early adopters including Codex, Vercel, Cline, and Cursor are already reporting measurable latency reductions of 30–40% in production coding agents.
