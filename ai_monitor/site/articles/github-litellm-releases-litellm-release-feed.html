LiteLLM v1.83.11-nightly Drops with Prompt Compression, Scaleway Audio Support, and Mantle Endpoint

A nightly release brings proxy-side prompt compression, Scaleway audio integration, and early Bedrock Mantle support to LiteLLM.

GitHub LiteLLM Releases 2026-04-22 12:43 UTC Key: github-litellm-releases::identity::github-litellm-releases-litellm-release-feed Confidence: strong Mode: claude

Article body

LiteLLM v1.83.11-nightly landed on 22 April 2026, the latest in the project's rapid nightly cadence. The release introduces prompt compression directly in the proxy layer, a new `--reload` flag for uvicorn hot reload during development, and audio support for Scaleway. Documentation was also added covering which providers support prompt caching.

On the infrastructure side, the release tackles a long-standing bug where `/health/readiness` could loop with 503 errors if the database was unreachable, and adds a freshness and destructive-action guard to the migration workflow. Bedrock users get a new Mantle endpoint with Claude Mythos Preview support, and the proxy gains an opt-in v2 migration resolver.

Several contributors made their first contributions in this release. All LiteLLM Docker images and release artifacts remain cryptographically signed via cosign, with verification instructions available using either a pinned commit hash or a release tag.

Why this matters

  • Prompt compression shipped into the proxy means developers can reduce token costs without changing client code.
  • The new Mantle endpoint (Claude Mythos Preview) on Bedrock extends LiteLLM's provider coverage to a newly surfaced AWS model.
  • The database unreachable health-check loop fix improves proxy reliability in containerised or shared-DB environments.
