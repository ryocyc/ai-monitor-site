Dify Adds Real-Time Collaboration and HITL API in Version 1.14.0

The open-source LLM app development platform Dify has shipped real-time multi-user workflow editing and a programmable human-in-the-loop service API, targeting teams that build and operate AI applications at scale.

Article body

Dify released version 1.14.0 on April 29, 2026, introducing a collaboration mode that allows multiple workspace members to edit the same workflow simultaneously, with synchronized graph updates, online presence indicators, and shared visibility into who is active where. The feature requires explicit opt-in on self-hosted deployments via three environment variables: ENABLE_COLLABORATION_MODE, SERVER_WORKER_CLASS set to GeventWebSocketWorker, and NEXT_PUBLIC_SOCKET_URL pointing to the deployment's WebSocket endpoint.

Also new in this release is a Human-in-the-Loop (HITL) Service API, which adds programmatic control over HITL flows beyond what the existing console UI provides. This enables teams to build custom review or approval pipelines into their AI workflows using code rather than relying solely on Dify's built-in interface. The release notes also reference MCP (Model Context Protocol) integration, though the full scope of that addition is not visible in the available excerpt.

Dify is an open-source platform for building, debugging, and deploying LLM-powered applications, commonly used to create AI agents, RAG pipelines, and multi-step reasoning workflows. The collaboration feature is most directly useful to teams that iterate on complex workflows together, such as prompt engineers, AI product teams, and MLOps groups managing production-grade AI deployments. The HITL API expands the platform's suitability for regulated or high-stakes environments where human oversight of model outputs is required before downstream actions are taken.

Why this matters

  • Real-time co-editing closes a key gap for teams that previously had to coordinate workflow changes sequentially or manage merge conflicts manually.
  • The HITL Service API enables programmatic human-in-the-loop guardrails, making Dify viable for use cases requiring audit trails, compliance reviews, or staged approvals in AI-driven processes.
  • The MCP integration signaled in the release notes suggests expanded model-context tooling, which would further broaden Dify's utility for agent-based architectures.
