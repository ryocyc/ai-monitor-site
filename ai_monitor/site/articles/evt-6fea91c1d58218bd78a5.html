Hugging Face Gradio Spaces Now Self-Describe for AI Agents Via /agents.md Endpoint

Every Gradio Space now automatically serves a machine-readable /agents.md endpoint, letting AI coding agents like Claude Code, Codex, and Pi discover and use spaces without manual configuration.

Hugging Face Changelog <span data-utc="2026-04-26T01:13:57+00:00">2026-04-26 01:13 UTC</span> Key: evt-6fea91c1d58218bd78a5 Confidence: strong Mode: claude

Article body

Hugging Face has added a self-describing interface to its Gradio Spaces platform: every space now automatically serves an /agents.md endpoint that AI agents can read on their own. The changelog entry, posted April 17, describes the endpoint as a machine-readable API description designed to let coding agents interact with spaces directly, with no setup required on either side.

When pointed at a space, an agent reads the /agents.md file and uses that description to figure out how to call the space's functions. The changelog specifically mentions Claude Code, Codex, and Pi as examples of agents that can work with this interface. The change applies across all Gradio Spaces hosted on huggingface.co/spaces.

For developers building AI-assisted coding tools, the endpoint removes a common friction point: integrating with a new space previously required custom wiring or brittle prompt engineering. Now an agent can self-service that discovery by reading the standardized description, potentially accelerating tool adoption across the thousands of spaces available on the platform.

Why this matters

  • Removes manual integration overhead for developers building AI coding agents by enabling self-service discovery of space capabilities.
  • Creates a standardized, machine-readable contract that any agent can follow, reducing per-space adapter work across the Hugging Face ecosystem.
  • Establishes a shared pattern that could become a reference approach for agent-to-tool discovery beyond Gradio.
