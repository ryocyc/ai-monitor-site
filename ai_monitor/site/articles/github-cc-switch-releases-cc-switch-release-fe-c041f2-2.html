cc-switch v3.14.1 Ships Tray Usage Markers and Codex OAuth Stability Improvements

The latest patch for the Claude/Codex/Gemini provider-switching tool surfaces cached usage directly in the system tray and addresses several Codex OAuth reverse-proxy reliability issues.

GitHub cc-switch Releases 2026-04-23 13:23 UTC Key: github-cc-switch-releases::identity::github-cc-switch-releases-cc-switch-release-fe Confidence: strong Mode: claude

Article body

CC Switch v3.14.1, released 23 April 2026, adds tray usage visibility to the application's system tray menu. Submenus for the active Claude, Codex, or Gemini provider now display cached usage data—including subscription summaries and usage-script summaries with color-coded utilization markers—and synchronize those readings back into React Query. For Chinese coding-plan hosts (Kimi, Zhipu, MiniMax), the tray renders a 5-hour plus weekly window layout in the h12% w80% style, with the emoji reflecting the worst utilization tier, matching the official subscription badge layout.

The release also addresses three distinct Codex OAuth reverse-proxy stability problems. Session IDs provided by the client are now used as both the prompt_cache_key and the Codex session header, reducing unnecessary cache churn. Non-streaming Anthropic clients now receive correct JSON responses even when the upstream ChatGPT Codex forces OpenAI Responses SSE. Stream Check probes now mirror production request settings—store: false, encrypted reasoning included, and provider FAST mode respected—eliminating false negatives where checks failed but actual requests succeeded. A new explicit FAST mode toggle lets users opt into service_tier="priority" on Codex OAuth-backed providers, trading latency for reduced ChatGPT quota consumption.

The in-app Hermes configuration health scanner and its associated warning banner have been removed, along with the scan_hermes_config_health command, HermesHealthWarning type, and HermesWriteOutcome.warnings payload. The Hermes surface is now focused on provider management, switching defaults, memory editing, and launching the Web UI. Configuration health for Hermes itself is now treated as Hermes's own responsibility. Skills import and install reliability also received targeted fixes in this release.

Why this matters

  • Tray-level usage visibility means users can monitor provider consumption without opening any modals or leaving their current context.
  • Codex OAuth stability fixes eliminate false Stream Check failures and improve cache efficiency for reverse-proxy setups using Claude-compatible endpoints backed by ChatGPT Codex.
  • Removal of the Hermes health scanner simplifies the interface and reduces noise from redundant configuration warnings, while the new FAST mode toggle gives users direct control over priority tier behavior on Codex-backed providers.
