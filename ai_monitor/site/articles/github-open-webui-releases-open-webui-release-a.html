Open WebUI Patches Startup Crash by Restoring Missing Database Dependencies

Open WebUI version 0.9.1 fixes a startup crash affecting users who installed the tool via pip or uv, caused by two dependencies that were missing from the published package metadata.

GitHub Open WebUI Releases 2026-04-22 15:01 UTC Key: github-open-webui-releases::identity::github-open-webui-releases-open-webui-release-a Confidence: strong Mode: claude

Article body

Open WebUI version 0.9.1, released on April 21, 2026, closes a startup crash that struck users who installed the open-source AI interface through pip or uv. The crash—triggered by a ModuleNotFoundError for the module named 'aiosqlite'—stemmed from that package being listed in requirements.txt but absent from the published package's pyproject.toml metadata, meaning it was never automatically installed alongside Open WebUI.

The same release also adds the asyncpg package to pyproject.toml, preventing an analogous crash for users running Open WebUI against a PostgreSQL database. Like aiosqlite, asyncpg was present in the project's requirements.txt but omitted from the package metadata that pip and uv consult during installation.

The Open WebUI team notes this release includes database schema changes and strongly advises backing up all database data before upgrading in production environments. A companion release, v0.9.0, arrived on the same day introducing a native desktop app for Mac, Windows, and Linux, a calendar workspace with event management and scheduling capabilities, and numerous performance improvements across async database operations, streaming responses, and memory handling.

Why this matters

  • Users installing Open WebUI via pip or uv—rather than Docker—would encounter an immediate crash on first launch.
  • PostgreSQL deployments were independently affected by the same metadata gap via the missing asyncpg dependency.
  • The same April 21 release also delivered a native desktop app, calendar workspace, and large-scale async performance improvements.
