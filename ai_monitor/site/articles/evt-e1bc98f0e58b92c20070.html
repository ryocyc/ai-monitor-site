llama.cpp Adds Native Blackwell NVFP4 Support in Latest Pre-built Binaries

The open-source llama.cpp project released build b8967 with native NVFP4 tensor support for NVIDIA Blackwell GPUs, expanding its already broad portfolio of pre-built binaries across macOS, Linux, Windows, iOS, and Android.

Article body

The ggml-org/llama.cpp project published release tag b8967 on 29 April 2026, adding native Blackwell-native NVFP4 (NVIDIA 4-bit floating point) tensor support under the CUDA backend via PR #22196. The release ships pre-built binaries for an unusually wide range of platforms and architectures, including macOS (Apple Silicon with KleidiAI enabled, and Intel x64), iOS as an XCFramework, multiple Linux distributions and CPU/Vulkan/ROCm/OpenVINO/SYCL variants, Android arm64, and Windows with CPU, CUDA 12, CUDA 13, Vulkan, SYCL, and HIP backends.

llama.cpp is a lightweight, dependency-free C/C++ inference engine and tensor library (ggml) for running large language models locally on commodity hardware. It sits at the foundation of a large ecosystem of tools — including LM Studio, Ollama front-ends, and a wide range of quantized model formats — and is widely used by developers who need to run models without relying on a hosted API. The NVFP4 addition targets inference engineers and platform engineers working with NVIDIA's Blackwell architecture who need finer-grained control over numeric precision on newer GPU silicon.

Practically, the Blackwell NVFP4 support means that applications built on top of llama.cpp can now target the H200 and Blackwell GPU line with 4-bit floating point weights, potentially improving throughput and memory efficiency for large model inference on that hardware. For developers distributing pre-built llama.cpp binaries, the expanded backend matrix in b8967 reduces the need for users to compile from source on less common platform configurations.

Why this matters

  • Blackwell NVFP4 support enables more memory-efficient inference on NVIDIA's latest GPU architecture, directly benefiting teams deploying large models on-premise or at edge.
  • The breadth of pre-built binary targets — spanning iOS, Android, macOS, Windows, and multiple Linux hardware configurations — makes it easier for application developers to distribute llama.cpp-powered features without a custom compilation step.
  • As a foundational inference layer for the broader local LLM ecosystem (Ollama, LM Studio, and numerous quantized model formats), changes to llama.cpp propagate widely and quickly into end-user products.
