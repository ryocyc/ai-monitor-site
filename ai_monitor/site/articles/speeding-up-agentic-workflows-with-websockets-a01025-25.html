OpenAI Adds WebSocket Support to Responses API, Cutting Agentic Workflow Latency by 40%

OpenAI has introduced WebSocket mode to its Responses API, enabling persistent connections that eliminate repeated API overhead and dramatically speed up agentic loops used by coding assistants and other AI agents.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When OpenAI's latest coding model, GPT-5.3-Codex-Spark, hit inference speeds of over 1,000 tokens per second—compared to roughly 65 TPS on previous flagship models—the cumulative overhead of the Responses API itself became the new bottleneck. Even after a 45% improvement in time-to-first-token through optimizations like caching and eliminating intermediate network hops, the API layer was still too heavy relative to how fast the model was now running. The root problem was structural: every follow-up request re-processed the full conversation history, regardless of how little had changed.

The solution OpenAI settled on was WebSocket-based persistent connections. Rather than establishing a fresh HTTP connection and sending the entire conversation context each time, the server maintains an in-memory cache of previous response state—including prior input/output items, tool definitions, and previously rendered tokens. When a follow-up request includes a `previous_response_id`, the API retrieves that cached state instead of rebuilding it from scratch. Critically, developers keep the same `response.create` input shape they already use, making the upgrade path incremental rather than a rewrite.

The performance gains have been validated across the developer ecosystem. Codex migrated the majority of its Responses API traffic to WebSocket mode within days of launch, and models including GPT-5.3-Codex, GPT-5.4, and later versions all benefit automatically. Vercel's AI SDK and Cline's multi-file workflows each reported roughly 40% latency reductions, while Cursor saw OpenAI model responses become up to 30% faster. OpenAI describes WebSocket mode as one of the most significant new capabilities in the Responses API since its March 2025 debut.

Why this matters

  • As AI model inference speeds continue to improve, the surrounding API infrastructure must keep pace—OpenAI identified API overhead as the new bottleneck once inference reached 1,000+ TPS.
  • The WebSocket approach lets developers keep their existing `response.create` integration intact while gaining the performance benefits of persistent connections, avoiding a costly migration.
  • Third-party integrations including Vercel, Cline, and Cursor have already reported measurable latency reductions of 30–40% in real production workflows, signaling broad applicability beyond OpenAI's own Codex agent.
