OpenHands Adds Task List Panel, Planning Agent, Slash Commands, and Bitbucket Support

Two recent releases of the AI coding agent OpenHands introduce a visible agent task list, a dedicated planning agent, slash-command shortcuts, and support for Bitbucket DataCenter alongside several new models.

GitHub OpenHands Releases 2026-04-22 15:01 UTC Key: github-openhands-releases::identity::github-openhands-releases-openhands-release-ac Confidence: strong Mode: claude

Article body

OpenHands 1.5.0 (released 2026-03-11) added a Task List tab in the right panel that displays the agent's current task list with real-time status updates, giving users direct visibility into what the agent is working on at any moment. Also new in that release: a Planning Agent that helps the system reason about multi-step tasks before executing them, and a slash command menu accessible by typing '/' in the chat input to invoke available agent skills without leaving the conversation. A Git repository can now be attached or changed for existing conversations, which addresses a long-standing workflow gap for users managing multi-repo projects.

Bitbucket DataCenter support landed in the same release, courtesy of a contribution by @phymbert. This expands OpenHands beyond its prior GitHub and GitLab integrations, opening the door for enterprise teams that run Bitbucket behind a firewall. The release also added support for a raft of new models including Claude Opus 4.6, GLM-4.7, Claude-Sonnet-4-6, GLM-5, Kimi-K2.5, Qwen3-Coder-Next, and Gemini-3.1-Pro-Preview. The microagent management UI was hidden in this release, which the team described as a cleanup of developer-facing controls.

OpenHands 1.6.0 (2026-03-30) followed two and a half weeks later with a /clear command for V1 conversations that starts a fresh chat history while retaining the current sandbox and configuration, a copy button on code blocks, global skill toggles, and Catalan language support. Multiple CVE fixes landed across both releases through dependency updates, and the team resolved a bug where conversations were occasionally not persisted and a bug in the Export Conversation feature that duplicated events multiple times.

Why this matters

  • The Task List panel gives users an unprecedented window into the agent's reasoning and progress, addressing a key opacity complaint with autonomous coding agents.
  • Bitbucket DataCenter support unlocks OpenHands for enterprise teams that cannot use hosted GitHub or GitLab, significantly expanding its addressable user base.
  • The slash command menu and /clear command make the interface more ergonomic for power users who rely on keyboard-driven workflows, while the new model support keeps OpenHands competitive with frontier model releases.
