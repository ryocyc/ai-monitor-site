Grok Speech to Text and Text to Speech APIs

xAI has released two standalone audio APIs — Grok Speech to Text and Grok Text to Speech — built on the infrastructure powering Grok Voice, Tesla vehicles, and Starlink customer support.

xAI Blog 2026-04-22 23:49 UTC Key: xai-blog::identity::grok-stt-and-tts-apis Confidence: strong Mode: claude

Article body

The new Grok Speech to Text API offers both batch and real-time streaming modes via REST and WebSocket endpoints. It touts word-level timestamps, speaker diarization, multichannel audio support, and Inverse Text Normalization that converts raw spoken output into structured data — converting spoken numbers, dates, and currencies into proper formatted text. xAI backs these claims with benchmarks showing an overall Word Error Rate of 6.9%, compared to 9.0% for ElevenLabs, 11.0% for Deepgram, and 12.9% for AssemblyAI across phone call entities, video/podcasts, meetings, and telephone domains. The STT API supports 25 or more languages with seamless switching.

The Grok Text to Speech API produces natural, expressive voice output using simple inline speech tags — such as `[laugh]`, `[sigh]`, `[whisper]`, and semantic tags for emphasis and clarity — allowing developers to shape prosody and emotion without complex markup. Both APIs are available via xAI's developer console and documentation.

Pricing is straightforward: STT costs $0.10 per audio hour for batch transcription and $0.20 per audio hour for streaming, while TTS is priced at $4.20 per one million characters. No hidden fees or tiered complexity is mentioned, with full rate limits listed in the xAI API console.

Why this matters

  • Grok STT posts competitive Word Error Rates against established speech vendors, making xAI a credible entrant in the crowded transcription market.
  • The inline speech tags in TTS offer a developer-friendly way to add prosody and emotion without designing complex audio pipelines.
  • Bundling multilingual support, diarization, and Inverse Text Normalization into the base API reduces the number of services developers need to chain together.
