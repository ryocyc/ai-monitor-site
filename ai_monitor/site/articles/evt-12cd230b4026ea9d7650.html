llama.cpp Patch Release Fixes RPC Server Cache on Windows, Updates Logging

The ggml-org/llama.cpp inference runtime shipped a bug-fix release, b8949, resolving a cache failure on Windows in the RPC server and removing a conditional compilation guard around a logging call.

Article body

llama.cpp tagged release b8949 on 2026-04-27, closing at least one confirmed bug in its RPC server component. According to the release notes, a cache mechanism in the RPC server could silently fail on Windows environments (tracked as issue #22394), and a co-authored patch from contributor Kotaro added explicit directory creation and cache file-name logging while removing a GGML_LOG_INFO conditional compilation block.

The release ships pre-built binaries across a wide platform surface: macOS (Apple Silicon arm64, KleidiAI-enabled arm64, Intel x64) and iOS XCFramework; Linux builds for Ubuntu on x64 and arm64 with CPU, Vulkan, ROCm 7.2, OpenVINO, and SYCL targets plus an s390x CPU build; Android arm64 CPU; and Windows x64 and arm64 CPU builds alongside CUDA 12 (12.4 DLLs) and CUDA 13 (13.1 DLLs) variants.

For developers running local inference or building embedded LLM tooling, the RPC server cache fix is directly relevant to anyone using llama.cpp in a distributed or offloaded inference configuration on Windows, a platform that is less commonly tested in the project's CI. Removing the conditional GGML_LOG_INFO guard improves observability for builds that depend on that log level.

Why this matters

  • The RPC server cache bug could silently corrupt or bypass caching on Windows, leading to unexpected memory usage or performance regressions in distributed llama.cpp deployments.
  • Broader CUDA coverage with both CUDA 12.4 and 13.1 DLL variants signals continued maintenance of Nvidia GPU acceleration paths as the ecosystem matures.
  • Cross-platform binary availability across Linux, macOS, Windows, iOS, and Android makes llama.cpp one of the most portable inference runtimes available, directly serving developers building on-device or edge AI products.
