last30days-skill v3.1.0 Integrates X API v2 OAuth2 and Adds Competitor Fan-Out Flag

The last30days-skill tool, used for tracking and comparing entities over a rolling 30-day window, shipped v3.1.0 with a new official X API v2 backend via OAuth2 and a --competitors flag for automated comparison discovery.

GitHub last30days-skill Releases 2026-04-23 05:11 UTC Key: github-last30days-skill-releases::identity::github-last30days-skill-releas-last30days-skill-rel Confidence: moderate Mode: claude

Article body

The release consolidates changes from versions 3.0.10 through 3.0.14, grouping more than a dozen pull requests into a single milestone. Among the most practically significant additions is a new xurl CLI that serves as an alternative X (Twitter) search backend, built on the official X API v2 using OAuth2 authentication. This gives users a supported path to query X data without relying on unofficial scraping endpoints.

The --competitors flag introduces automatic discovery of comparison targets, allowing the tool to fan out analysis across competing entities without requiring manual specification each run. This pairs with a new "vs mode N" that performs multiple full passes over the dataset. Evidence lines can now also attribute top comments with u/ and @ handles, making it easier to trace which Reddit users or Twitter accounts contributed to a given data point.

On the configuration side, the memory directory is now configurable (contributed by new contributor @davemorin), and several under-the-hood fixes address subreddit expansion logic, planner breadth, and entity grounding for Hermes Agent use cases. Two new first-time contributors — @davemorin and @chaosreload — had their first commits merged in this release.

Why this matters

  • The official X API v2 OAuth2 backend replaces a potentially fragile unofficial search path, improving reliability for users tracking X data.
  • The --competitors flag automates the discovery of comparison targets, saving manual setup time for competitive analysis workflows.
  • Configurable memory directories allow users to scope where the tool stores its working data, a practical improvement for multi-project environments.
