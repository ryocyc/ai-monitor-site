GitHub Releases Codex 0.125.0 with Unix Socket Transport, Remote Plugin Support, and Rollout Tracing

OpenAI shipped a new stable release of its coding agent, GitHub Codex, featuring app-server integrations for Unix sockets, remote plugin installation, and expanded rollout tracing across multi-agent sessions.

Article body

OpenAI published GitHub Codex version 0.125.0 on April 24, 2026, the latest stable release for its AI-powered coding agent. The update introduces several infrastructure-level improvements to the app-server integration layer, including Unix socket transport support, pagination-friendly resume and fork operations, sticky environments, and remote thread configuration and store plumbing. These additions are designed to give enterprise and self-hosted users more flexibility in how Codex connects to backend services.

The release also brings remote plugin management capabilities, allowing users to install and upgrade plugins from remote sources and configured marketplaces. Permission profiles now persist correctly across TUI sessions, user turns, MCP sandbox state, shell escalation, and app-server APIs, resolving a long-standing consistency issue. Additionally, model providers now own model discovery, with AWS and Bedrock account state exposed directly to application clients, streamlining multi-cloud setups. For programmatic consumers, `codex exec --json` now reports reasoning-token usage.

Rollout tracing has been expanded to record tool, code-mode, session, and multi-agent relationships, with a new debug reducer command for inspection. On the bug-fix side, interrupting `/review` and exiting the TUI no longer wedges the interface, and the exec-server no longer drops buffered output after process exit.

Why this matters

  • Unix socket transport and sticky environments make Codex more suitable for production and containerized deployment scenarios where network configuration is constrained.
  • Remote plugin management and AWS/Bedrock integration widen the paths for enterprise users to customize and extend Codex within their existing cloud infrastructure.
  • Rollout tracing improvements give developers and platform teams better visibility into how Codex executes across tools, sessions, and multi-agent contexts.
