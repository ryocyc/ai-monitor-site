Open WebUI v0.9.1 Fixes Two Missing Dependencies a Day After v0.9.0

Open WebUI pushed a quick patch on April 21 to resolve startup crashes caused by two missing Python package dependencies in the published package metadata.

GitHub Open WebUI Releases 2026-04-22 12:43 UTC Key: github-open-webui-releases::identity::github-open-webui-releases-open-webui-release-f Confidence: strong Mode: claude

Article body

Open WebUI released version 0.9.1 on April 21, just hours after the v0.9.0 rollout. The patch addresses two missing Python package dependencies—aiosqlite and asyncpg—that caused ModuleNotFoundError crashes at startup for users who installed the software via pip or uv. Both packages were listed in the project's requirements.txt file but were absent from pyproject.toml, meaning they were not included in the published package metadata and therefore not installed automatically.

Specifically, aiosqlite was needed for SQLite users and asyncpg for PostgreSQL users. Without them, Open WebUI would fail to launch with the error "ModuleNotFoundError: No module named 'aiosqlite'" regardless of the database backend. The fix adds both packages to pyproject.toml, aligning the published package dependencies with what was already present in requirements.txt. Pull request #23916 addresses both issues together.

The release drew 43 thumbs-up reactions and 14 rocket reactions from the community, indicating the fix resolved a pain point for a meaningful portion of the user base. The repository has over 134,000 stars, making it a widely used open-source project.

Why this matters

  • Users installing Open WebUI via pip or uv encountered immediate startup crashes due to the missing dependencies, making the software effectively unusable out of the box.
  • The fix directly targets installation workflows, which are the primary path for many self-hosted users who do not use Docker.
  • The rapid turnaround—patch released the same day as v0.9.0—suggests the issue was widely reported by users and prioritized accordingly.
