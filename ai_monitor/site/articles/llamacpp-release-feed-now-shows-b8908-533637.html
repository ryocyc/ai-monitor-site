llama.cpp Release b8908 Fixes High-Severity Heap-Buffer-Overflow in Server Component

llama.cpp has released version b8908 to address a heap-buffer-overflow vulnerability triggered when a client sends a negative n_discard value in JSON input.

GitHub llama.cpp Releases 2026-04-23 23:18 UTC Key: github-llamacpp-releases::identity::b8908 Confidence: strong Mode: claude

Article body

The vulnerability, tracked as CVE-2026-21869 and classified under CWE-787, resides in the update_slots() context-shift loop of the llama.cpp server. When a client sends a negative n_discard value in JSON, it causes memory corruption that can lead to a crash or potentially allow arbitrary code execution. The flaw carries a CVSS score of 8.8, placing it in the high-severity range.

The fix applies a clamping guard at the JSON parse boundary, enforcing a non-negative value for n_discard before it reaches the update_slots() function. According to the release notes, even n_discard=0 already triggers the auto-discard behavior (discarding n_left/2 tokens), so clamping to 0 is a safe and effective mitigation. The change also references GitHub Security Advisory GHSA-8947-pfff-2f3c.

Pre-built binaries for this release are available across a wide range of platforms: macOS (Apple Silicon with optional KleidiAI, and Intel), iOS (XCFramework), multiple Linux flavors including Ubuntu across CPU, Vulkan, ROCm 7.2, OpenVINO, and SYCL variants, Android arm64, and Windows with CUDA 12/13, Vulkan, SYCL, and HIP support. OpenEuler builds for both x86 and aarch64 are also included.

Why this matters

  • The vulnerability is remotely triggerable by any client sending crafted JSON to a llama.cpp server, making it a significant attack surface for any deployment exposed over a network.
  • With a CVSS score of 8.8, the flaw is serious enough to warrant immediate updating on any production llama.cpp server instance.
  • Pre-built binaries are available for a broad range of platforms, making the patch accessible to most users without requiring a manual build from source.
