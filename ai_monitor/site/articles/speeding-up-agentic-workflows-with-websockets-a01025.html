OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, cutting agentic workflow latency by up to 40 percent and enabling Codex to hit over 1,000 tokens per second in production.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When a coding agent like Codex fixes a bug, it fires dozens of back-and-forth API requests to scan code, make edits, and run tests—adding up to minutes of wait time for complex tasks. OpenAI's engineering team set out to eliminate that overhead, targeting a 40 percent speedup in end-to-end agent loops.

The bottleneck emerged as inference got faster. Flagship models like GPT-5 ran at roughly 65 tokens per second, making API service overhead easy to hide. But with the launch of GPT-5.3-Codex-Spark on specialized Cerebras hardware—hitting over 1,000 tokens per second—the API layer became the limiting factor. OpenAI's team cached tokens and model configuration, cut intermediate network hops, and sped up safety classifiers, landing a 45 percent improvement in time to first token. Still, the deeper problem was structural: each follow-up request re-processed the full conversation history even when little had changed.

The solution was to rethink the transport protocol itself. OpenAI considered bidirectional streaming options but settled on WebSockets, which let developers keep the familiar `response.create` shape and a `previous_response_id` parameter while the server maintains an in-memory cache of prior response state. The approach eliminated redundant API work across an entire agent rollout, allowing preinference work to run once, pause for tool execution, and complete postinference once at the end. After a two-month alpha with coding agent startups, Codex ramped the majority of its Responses API traffic onto WebSockets, reaching bursts of 4,000 tokens per second and proving the API could keep pace with much faster inference hardware.

Why this matters

  • OpenAI reports up to 40% latency improvements for agentic workflows in production, with Codex users on GPT-5.3-Codex and later models automatically benefiting.
  • Third-party integrations—including Vercel AI SDK, Cline, and Cursor—show consistent speedups of 30–39% for multi-file or multi-turn coding tasks.
  • The design proves that as inference hardware speeds up, the API and transport layers must be redesigned in parallel to transfer those gains to end users.
