LangGraph Prebuilt Package Updated to 1.0.12 with Core Engine Upgrades

LangGraph's production-ready prebuilt package has been bumped to version 1.0.12, bringing updated internal dependencies and a fix for how ToolNode handles agent state hydration.

Article body

LangChain AI has released langgraph-prebuilt version 1.0.12, the latest iteration of its opinionated, deployment-ready packaging of the LangGraph framework for building stateful multi-actor LLM applications. The release landed on April 27, 2026.

The update bundles three coordinated changes: the core LangGraph runtime is bumped to 1.1.10, the checkpointing subsystem is updated to 4.0.3, and a bug fix addresses how ToolNode — the component that lets agents invoke tools — hydrates state from underlying channel data using Pregel helpers. All three changes reference specific pull requests (#7623, #7625, #7594), indicating a coordinated minor release rather than a hotfix.

For agent builders deploying LangGraph in production, the ToolNode hydration fix is the most directly impactful change. Incorrect state hydration could cause tools to operate on stale or incomplete agent context, a class of bug that is hard to diagnose in complex multi-step workflows. The checkpoint and runtime bumps ensure teams running the prebuilt package are not left behind on older internal dependencies.

Why this matters

  • The ToolNode state hydration fix resolves a potential source of silent failures in multi-step agentic pipelines where tool results feed back into agent memory.
  • The prebuilt package is the recommended production entry point for LangGraph, so dependency bumps here signal that the ecosystem is moving toward LangGraph 1.1.x and checkpoint 4.x.
  • Agent builders using the prebuilt image or distribution can patch by upgrading their dependency without rewriting any workflow code.
