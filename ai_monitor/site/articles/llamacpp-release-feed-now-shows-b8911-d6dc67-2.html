llama.cpp b8911 Revamps Hexagon DSP Profiling Infrastructure

Release b8911 of llama.cpp delivers a sweeping set of changes to its Hexagon DSP backend, centering on a rebuilt profiling system with async operation tracking, optional PMU counters, and a new post-processing tool.

GitHub llama.cpp Releases 2026-04-24 01:24 UTC Key: github-llamacpp-releases::identity::b8911 Confidence: strong Mode: claude

Article body

The GitHub Actions release, tagged b8911 (commit 5d2b52d, signed GitHub GPG key B5690EEEBB952194), lands on 24 April 2026 and targets the Hexagon DSP backend used in Qualcomm-powered devices. The headline change is a refactored profiler: operations previously guarded by an "opsync" mechanism are now fully asynchronous, and the old "opmask" label has been replaced with the more accurate "opstage" to reflect how queue elements are tracked through execution stages rather than masking behavior.

A key practical change is making performance monitoring unit (PMU) counters optional on older hardware, where retrieving them was found to be expensive. Users can now set which PMU events to collect via an environment variable and tune profiler mode to suit their device. The release also introduces a lightweight post-processing script (`scripts/snapdragon/ggml-hexagon-profile.py`) that accepts logs from stdin, and the `GGML_HEXAGON_PROFILE` environment variable is now formally documented.

Several supporting refinements round out the release: op-queuing logic is simplified with separate profile descriptors, an opbatch queue handler is refactored, host-facing interface hooks are added to toggle the profiler from outside the device, and linting issues in the Hexagon ops header are cleaned up. Co-authorship includes Sigbjørn Skjæret and Trivikram Reddy.

Why this matters

  • Developers running llama.cpp on Qualcomm Hexagon hardware — common in Android devices and always-on AI profiles — now get a measurably better debugging story, with lower overhead on constrained hardware.
  • The switch from opsync/async to fully async profiling, plus the renamed opstage, signals a maturing of the Hexagon backend that could feed into broader stability and correctness improvements.
  • The new post-proc tool and documented `GGML_HEXAGON_PROFILE` env var make it practical for non-experts to profile neural network kernels on-device for the first time.
