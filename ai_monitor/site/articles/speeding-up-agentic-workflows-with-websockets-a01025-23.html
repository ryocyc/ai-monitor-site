OpenAI Responses API Now Supports WebSockets for Faster Agentic Workflows

OpenAI has added WebSocket support to its Responses API, enabling persistent connections that reduce agentic workflow latency by up to 40 percent.

OpenAI Newsroom 2026-04-22 21:57 UTC Key: openai-newsroom::identity::speeding-up-agentic-workflows-with-websockets Confidence: strong Mode: claude

Article body

When Codex fixes a bug, it typically fires dozens of back-and-forth requests to the Responses API — determine the next action, run a tool, send results back, and repeat. OpenAI's engineering team found that as model inference speeds increased, the cumulative overhead of those API calls became the new bottleneck. The solution was a WebSocket mode that keeps a persistent connection open, caching conversation state server-side so each follow-up request only sends new information rather than rebuilding the full context from scratch.

The persistent WebSocket connection lets the API maintain an in-memory cache of prior response objects, input and output items, tool definitions, and previously rendered tokens. When a follow-up request includes a `previous_response_id`, the API retrieves that cached state instantly instead of recomputing it — reducing redundant tokenization, safety checks, and routing logic on every turn. This design was deliberately kept familiar: developers still use the standard `response.create` body, just over a live WebSocket connection rather than a series of independent HTTP calls.

The results validated the approach. GPT-5.3-Codex-Spark, built on specialized Cerebras hardware, hit the 1,000 tokens-per-second target (with bursts up to 4,000 TPS) — compared to roughly 65 TPS for previous flagship models. Alpha testers reported up to 40% improvements in agentic workflow latency. After launch, Codex quickly migrated most Responses API traffic to WebSocket mode, and integrations with Vercel AI SDK (~40% reduction), Cline (39% faster), and Cursor (30% faster) extended the gains to a wider developer base.

Why this matters

  • As inference hardware gets faster, API overhead becomes the limiting factor for agentic speed — WebSockets close that gap by eliminating repeated per-request processing.
  • Developers can adopt WebSocket mode without changing their Responses API input and output shapes, lowering the integration barrier.
  • The speed gains apply broadly: Codex, Cursor, Cline, and Vercel AI SDK users all report significant latency reductions in production agent workflows.
