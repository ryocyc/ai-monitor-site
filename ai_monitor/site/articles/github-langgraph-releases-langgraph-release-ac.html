LangGraph v1.1.9 Patches ReplayState Propagation and Stream Handler Bugs

LangGraph 1.1.9 resolves a regression where ReplayState was incorrectly propagated to subgraphs on plain resume, and undoes an unintended change in the stream handler.

GitHub LangGraph Releases 2026-04-22 15:01 UTC Key: github-langgraph-releases::identity::github-langgraph-releases-langgraph-release-ac Confidence: moderate Mode: claude

Article body

The LangGraph team has released version 1.1.9, addressing two issues introduced in recent releases. The primary fix stops ReplayState from being propagated to subgraphs during a plain resume operation, a bug that could cause unexpected behavior in graph traversal. A second fix reverts unnecessary changes made to the stream handler that had no functional benefit.

The prior release, v1.1.8, removed a strict `add_handler` type check that was known to break OpenTelemetry (OTel) instrumentation setups. That change was shipped alongside v1.1.7, which had itself addressed time-travel behavior when navigating back to an interrupt node. The v1.1.7 release also bumped the langsmith dependency from 0.6.4 to 0.7.31 across the library.

Supporting packages received updates in the same window. The CLI advanced to 0.4.24, the prebuilt library to 1.0.10 (fixing handling of injected NotRequired keys), and the checkpoint library to 4.0.2, which added documentation for the LANGGRAPH_STRICT_MSGPACK environment variable.

Why this matters

  • ReplayState propagation to subgraphs could silently corrupt graph state during resume operations, making the fix critical for anyone using interruption and replay patterns.
  • The OpenTelemetry fix in v1.1.8 confirms LangGraph is actively addressing observability tooling compatibility, which matters for teams running distributed tracing.
  • The parallel release of CLI, prebuilt, and checkpoint packages means downstream users of the LangGraph ecosystem should update those libraries in tandem.
